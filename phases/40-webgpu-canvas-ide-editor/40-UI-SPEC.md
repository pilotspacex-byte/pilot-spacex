# Phase 40 -- UI Design Contract

> Visual and interaction contract for the Monaco Editor migration and IDE file browser. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tab close button padding |
| sm | 8px | Compact element spacing, file tree node indent, tab bar item gap |
| md | 16px | Default element spacing, editor gutter width, panel padding |
| lg | 24px | Section padding, file tree header padding |
| xl | 32px | Layout gaps between panels (file tree / editor / preview) |
| 2xl | 48px | Quick Open dialog vertical offset from top |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Tab bar height: 36px (compact, consistent with VS Code convention)
- File tree row height: 28px (dense tree, fits more items in viewport)
- Quick Open input height: 40px (comfortable Cmd+P search input)
- View zone minimum height: 80px (PM block collapsed state)

## Interaction Contracts

### Tab Bar

- Maximum visible tabs: fill available width, then overflow into a `...` dropdown (Popover with list)
- Tab close: `X` icon appears on hover (not always visible). Middle-click also closes.
- Active tab: 2px bottom border in `--primary` color. Text in `--foreground`.
- Inactive tab: No bottom border. Text in `--muted-foreground`.
- Tab order: insertion order (most recently opened on the right)
- Dirty indicator: dot before file name in `--primary` color when unsaved changes exist
- Double-click tab: (no action -- reserved for future "pin" feature)

### File Tree

- Indent per level: 16px (md spacing)
- Expand/collapse: chevron icon rotates 90 degrees with 150ms ease-out transition
- Selected file: `--background-muted` background, `--primary` 2px left border
- Hover: `--background-subtle` background
- Context menu (right-click): Open, Open to the Side, Copy Path, Reveal in Finder (Tauri only)
- File icons: Lucide icons mapped by file extension (folder, file-text, file-code, file-image, etc.)
- Virtualized via `react-virtuoso` for directories with 500+ items

### Quick Open (Cmd+P)

- Overlay: centered horizontally, 48px from top of viewport
- Width: 560px max, responsive down to 100% minus 32px padding on small screens
- Input: 40px height, `--font-mono` for path display, auto-focus on open
- Results: max 10 visible, virtualized list, keyboard arrow navigation
- Match highlighting: matched characters in `--primary` color, bold weight
- File path shown in `--muted-foreground` below file name
- Dismiss: Escape key or click outside

### Editor Viewport

- Gutter (line numbers): visible, `--muted-foreground` text, 16px right padding
- Minimap: hidden by default (space premium in IDE layout)
- Word wrap: on for markdown/note files, off for code files
- Cursor: `--primary` color, line cursor (not block)
- Selection: `--primary-muted` background
- Ghost text: `--ai` color, italic style, Tab to accept, Escape to dismiss
- Scrollbar: thin (7px), `--border` thumb color, auto-hide after 1.5s inactivity

### Markdown Preview Toggle

- Toggle button in EditorToolbar: "Edit" / "Preview" text labels (not icons)
- Crossfade transition: 200ms opacity transition between edit and preview modes
- Preview container: max-width 720px, centered, 32px horizontal padding
- Preview typography: `--font-sans` for prose, `--font-mono` for code blocks
- Code blocks in preview: `--background-subtle` background, 14px `--font-mono`, 8px padding, `--radius-md` corners
- Mermaid diagrams: rendered inline via existing `MermaidPreview` component
- KaTeX math: inline and block rendering with `katex.min.css` styles

### PM Block View Zones

- View zone background: `--background-subtle` with `--border-subtle` 1px top/bottom border
- View zone padding: 16px all sides
- View zone minimum height: 80px (collapsed state)
- Expand/collapse: chevron toggle, 200ms height transition
- View zone renderers: plain React components (NOT observer-wrapped -- React 19 flushSync constraint)
- Data flow: Monaco editor context provider passes block data to renderers

### Panel Layout

- Three-panel resizable layout: File Tree (left) | Editor (center) | Preview (right, conditional)
- File tree default width: 240px, collapsible to 0px, max 400px
- Editor: fills remaining space, minimum 400px
- Preview panel: appears only in preview mode, replaces editor (toggle, not side-by-side)
- Panel resize handles: 4px wide, `--border` color, cursor: col-resize
- Crossfade on panel show/hide: 200ms opacity + 200ms width transition

### Spring Physics Scroll (Lenis)

- Applied to: file tree scroll container, note list sidebar
- NOT applied to: Monaco editor viewport (Monaco handles own scroll), modals, settings
- `data-lenis-prevent` attribute on all Monaco editor DOM containers
- Lerp: 0.1, duration: 1.2s, smooth wheel enabled, touch multiplier 2
- Deceleration: natural momentum with smooth stop (no hard cut)

## Accessibility Contract

- All interactive elements keyboard-reachable (Tab order: sidebar > tab bar > editor > toolbar)
- File tree: arrow key navigation (Up/Down to move, Right to expand, Left to collapse, Enter to open)
- Quick Open: arrow key navigation for results, Enter to select, Escape to close
- Tab bar: Ctrl+Tab / Ctrl+Shift+Tab to cycle tabs (matches browser convention)
- Editor: full Monaco keyboard accessibility (built-in)
- Focus ring: 2px `--ring` (#29a386) outline on all focusable elements, visible on keyboard focus only (`:focus-visible`)
- Color never sole meaning carrier: dirty indicator uses dot + tooltip text, not just color
- ARIA: file tree uses `role="tree"` / `role="treeitem"`, tabs use `role="tablist"` / `role="tab"` / `role="tabpanel"`
- Reduced motion: respect `prefers-reduced-motion` -- disable spring scroll, use instant transitions
