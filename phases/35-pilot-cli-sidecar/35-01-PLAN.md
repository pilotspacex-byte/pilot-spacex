3. Create `cli/pilot.spec` — a PyInstaller spec file using `--onedir` mode (NOT `--onefile` — per CONTEXT.md, `--onefile` is incompatible with Tauri sidecar on Windows):

```python
# -*- mode: python ; coding: utf-8 -*-
"""PyInstaller spec for pilot-cli sidecar binary.

Build: pyinstaller cli/pilot.spec --noconfirm
Output: dist/pilot-cli/ (onedir bundle)

The entry point binary is dist/pilot-cli/pilot-cli which Tauri
spawns as a sidecar. The entire dist/pilot-cli/ directory must be
copied to tauri-app/src-tauri/binaries/ with the main binary
renamed to pilot-cli-{target_triple}.
"""
import sys
from pathlib import Path

block_cipher = None

# Resolve paths relative to spec file location
cli_dir = Path(SPECPATH)
src_dir = cli_dir / 'src'

a = Analysis(
    [str(cli_dir / 'src' / 'pilot_cli' / 'main.py')],
    pathex=[str(src_dir)],
    binaries=[],
    datas=[
        # Include Jinja2 templates used by implement command
        (str(cli_dir / 'src' / 'pilot_cli' / 'templates'), 'pilot_cli/templates'),
    ],
    hiddenimports=[
        # Core CLI framework
        'typer',
        'typer.main',
        'typer.core',
        'click',
        'click.core',
        'click.decorators',
        # HTTP client
        'httpx',
        'httpx._transports',
        'httpx._transports.default',
        'httpcore',
        'h11',
        'anyio',
        'anyio._backends',
        'anyio._backends._asyncio',
        'sniffio',
        'certifi',
        'idna',
        # Rich console output
        'rich',
        'rich.console',
        'rich.text',
        'rich.markup',
        'rich.progress',
        'pygments',
        # Git operations
        'git',
        'gitdb',
        'smmap',
        # Jinja2 templating
        'jinja2',
        'jinja2.ext',
        'markupsafe',
        # TOML writing
        'tomli_w',
        # GitHub API
        'github',
        'github.MainClass',
        'urllib3',
        'nacl',  # PyNaCl used by PyGithub for encryption
        # Cryptography (used by various deps)
        'cryptography',
        'cryptography.hazmat',
        'cryptography.hazmat.primitives',
        'cryptography.hazmat.primitives.asymmetric',
        'cryptography.hazmat.backends',
        # Pilot CLI modules
        'pilot_cli',
        'pilot_cli.main',
        'pilot_cli.commands',
        'pilot_cli.commands.implement',
        'pilot_cli.commands.login',
        'pilot_cli.commands.backup',
        'pilot_cli.config',
        'pilot_cli.api_client',
        'pilot_cli.github_client',
    ],
    hookspath=[],
    hooksconfig={},
    runtime_hooks=[],
    excludes=[
        # Dev-only deps — not needed in binary
        'pytest',
        'pytest_asyncio',
        'respx',
        'mypy',
        'ruff',
        'pytest_cov',
        # Unused stdlib modules — reduce binary size
        'tkinter',
        'unittest',
        'test',
    ],
    win_no_prefer_redirects=False,
    win_private_assemblies=False,
    cipher=block_cipher,
    noarchive=False,
)

pyz = PYZ(a.pure, a.zipped_data, cipher=block_cipher)

exe = EXE(
    pyz,
    a.scripts,
    [],
    exclude_binaries=True,  # onedir mode — binaries go in COLLECT
    name='pilot-cli',
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=True,
    console=True,  # CLI tool — needs console
    disable_windowed_traceback=False,
    argv_emulation=False,
    target_arch=None,
    codesign_identity=None,
    entitlements_file=None,
)

coll = COLLECT(
    exe,
    a.binaries,
    a.zipfiles,
    a.datas,
    strip=False,
    upx=True,
    upx_exclude=[],
    name='pilot-cli',
)
```

Key design decisions:
- `onedir` mode produces `dist/pilot-cli/` directory with the main binary + supporting files
- Hidden imports cover all transitive deps that PyInstaller's static analysis misses (typer, httpx, rich, git, jinja2, PyGithub, cryptography)
- Jinja2 templates directory included as data files (required by implement command)
- Dev dependencies excluded to reduce binary size
- `console=True` because this is a CLI tool that writes to stdout/stderr

  <action>
Create `.github/workflows/pilot-cli-build.yml` — a GitHub Actions workflow that builds the pilot-cli sidecar binary for all 4 platform targets.

The workflow:
1. Triggers on push to main/develop/feat/tauri-* branches AND pull_request to main/develop (same pattern as tauri-build.yml)
2. Uses the same 4-runner matrix as tauri-build.yml (macos-latest, macos-13, ubuntu-22.04, windows-latest)
3. Each runner: install Python 3.12 via actions/setup-python, install uv, uv sync, install pyinstaller, run pyinstaller on cli/pilot.spec, rename the output binary with the Tauri platform triple suffix, upload as artifact
4. Artifacts are named consistently for consumption by tauri-build.yml

```yaml
name: Pilot CLI Sidecar Build

on:
  push:
    branches: [main, develop, 'feat/tauri-*']
  pull_request:
    branches: [main, develop]

concurrency:
  group: pilot-cli-${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  build:
    strategy:
      fail-fast: false
      matrix:
        include:
          - platform: macos-latest
            target_triple: aarch64-apple-darwin
            binary_ext: ""
          - platform: macos-13
            target_triple: x86_64-apple-darwin
            binary_ext: ""
          - platform: ubuntu-22.04
            target_triple: x86_64-unknown-linux-gnu
            binary_ext: ""
          - platform: windows-latest
            target_triple: x86_64-pc-windows-msvc
            binary_ext: ".exe"

    runs-on: ${{ matrix.platform }}
    name: Build pilot-cli (${{ matrix.target_triple }})
    timeout-minutes: 30

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Install Python 3.12
        uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install uv
        uses: astral-sh/setup-uv@v4

      - name: Install CLI dependencies
        working-directory: cli
        run: uv sync

      - name: Install PyInstaller
        working-directory: cli
        run: uv pip install pyinstaller>=6.19.0

      - name: Build pilot-cli binary
        working-directory: cli
        run: uv run pyinstaller pilot.spec --noconfirm --clean

      - name: Rename binary with platform triple
        shell: bash
        run: |
          SRC="cli/dist/pilot-cli/pilot-cli${{ matrix.binary_ext }}"
          DEST="tauri-app/src-tauri/binaries/pilot-cli-${{ matrix.target_triple }}${{ matrix.binary_ext }}"
          mkdir -p tauri-app/src-tauri/binaries
          cp "$SRC" "$DEST"
          echo "Sidecar binary: $DEST"
          ls -la "$DEST"

      - name: Copy onedir support files
        shell: bash
        run: |
          # Copy the entire onedir bundle alongside the renamed binary
          # Tauri sidecar needs these support files next to the binary
          BUNDLE_DIR="tauri-app/src-tauri/binaries/pilot-cli-${{ matrix.target_triple }}-bundle"
          mkdir -p "$BUNDLE_DIR"
          cp -r cli/dist/pilot-cli/* "$BUNDLE_DIR/"
          # Remove the original binary from bundle dir (already copied with triple name)
          rm -f "$BUNDLE_DIR/pilot-cli${{ matrix.binary_ext }}"
          echo "Support files:"
          ls "$BUNDLE_DIR/" | head -20

      - name: Upload sidecar binary artifact
        uses: actions/upload-artifact@v4
        with:
          name: pilot-cli-${{ matrix.target_triple }}
          path: |
            tauri-app/src-tauri/binaries/pilot-cli-${{ matrix.target_triple }}${{ matrix.binary_ext }}
            tauri-app/src-tauri/binaries/pilot-cli-${{ matrix.target_triple }}-bundle/
          retention-days: 7
          if-no-files-found: error

      - name: Verify binary runs
        shell: bash
        run: |
          BINARY="tauri-app/src-tauri/binaries/pilot-cli-${{ matrix.target_triple }}${{ matrix.binary_ext }}"
          chmod +x "$BINARY" 2>/dev/null || true
          "$BINARY" --help
```

Key design decisions:
- Same branch triggers and concurrency pattern as tauri-build.yml for consistency
- `fail-fast: false` matches tauri-build.yml — all platform builds run to completion
- `uv run pyinstaller` ensures PyInstaller runs within the uv-managed virtual environment
- Binary renamed from `pilot-cli` to `pilot-cli-{target_triple}` per Tauri sidecar naming convention
- The onedir support files (shared libs, Python modules) are uploaded alongside in a `-bundle` directory
- `--help` smoke test verifies the binary actually runs on each platform
- timeout-minutes: 30 (PyInstaller builds are typically 2-5 minutes; generous margin)
- Artifact naming `pilot-cli-{target_triple}` for easy identification and consumption by tauri-build.yml
