In `.github/workflows/tauri-build.yml`, add a sidecar download step inside the `build` job, after the "Install Tauri CLI dependencies" step (line 78) and before the "Sign sidecar binary (macOS)" step (line 80).

The key design constraint: `tauri-build.yml` and `pilot-cli-build.yml` trigger on the same events (push to main/develop/feat/tauri-*) and run in parallel — we cannot use `needs:` to depend on `pilot-cli-build.yml` from `tauri-build.yml`. Instead use `dawidd6/action-download-artifact@v6` which can download artifacts from a different workflow run on the same commit SHA.

Insert this step block between "Install Tauri CLI dependencies" and "Sign sidecar binary (macOS)":

```yaml
      - name: Download sidecar artifact (pilot-cli)
        uses: dawidd6/action-download-artifact@v6
        with:
          workflow: pilot-cli-build.yml
          commit: ${{ github.sha }}
          name: pilot-cli-${{ matrix.target }}
          path: tauri-app/src-tauri/binaries/
          search_artifacts: true
          if_no_artifact_found: warn
```

Explanation of each parameter:
- `workflow: pilot-cli-build.yml` — targets the CLI build workflow by filename
- `commit: ${{ github.sha }}` — same commit SHA, so we get the artifact from the parallel run of the same push event
- `name: pilot-cli-${{ matrix.target }}` — matches the artifact name in pilot-cli-build.yml (line 93: `name: pilot-cli-${{ matrix.target_triple }}`, and matrix.target in tauri-build.yml matches target_triple values exactly)
- `path: tauri-app/src-tauri/binaries/` — destination where tauri-action expects to find externalBin
- `search_artifacts: true` — search across workflow runs on this commit if the exact run isn't indexed yet
- `if_no_artifact_found: warn` — warn (not fail) so the build proceeds with the stub binary when pilot-cli-build.yml hasn't completed yet; avoids a hard dependency that would deadlock parallel runs

This mirrors the tauri-release.yml pattern (lines 119-123) but uses dawidd6/action-download-artifact instead of actions/download-artifact@v4 because cross-workflow downloads require the extended action.

The step goes at this position in the YAML (after line 78, before line 80):

```yaml
      - name: Install Tauri CLI dependencies
        working-directory: tauri-app
        run: pnpm install --frozen-lockfile

      - name: Download sidecar artifact (pilot-cli)      # <-- INSERT HERE

```

Do not change any other step, env var, or job-level key. Only insert the one new step.
