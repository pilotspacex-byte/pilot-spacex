2. Create `frontend/src/stores/features/implement/ImplementStore.ts`:
   - Import types from `@/lib/tauri` (SidecarOutput, SidecarResult)
   - MobX observable fields:
     - `isRunning: boolean = false` — overall pipeline active
     - `currentStep: 'idle' | 'branching' | 'implementing' | 'staging' | 'committing' | 'pushing' | 'done' | 'error' = 'idle'`
     - `issueId: string | null = null` — current issue being implemented
     - `branchName: string | null = null` — auto-generated branch name
     - `repoPath: string | null = null` — path to the repo
     - `sidecarId: string | null = null` — running sidecar process ID for cancellation
     - `output: string[] = []` — collected sidecar output lines (max 500)
     - `error: string | null = null` — error message from any step
     - `commitOid: string | null = null` — resulting commit hash
   - MobX computed:
     - `get stepLabel(): string` — human-readable label for currentStep (e.g., "Creating branch...", "Running pilot implement...", "Staging changes...", "Committing...", "Pushing to remote...", "Complete!", "Failed")
     - `get canCancel(): boolean` — true when currentStep is 'implementing' (sidecar running)
   - MobX actions:
     - `startImplement(issueId: string, repoPath: string): Promise<void>` — the main orchestrator:
       a. Set isRunning=true, currentStep='branching', issueId, repoPath, clear error/output/commitOid
       b. Generate branch name: `implement/${issueId}` (e.g., `implement/PS-42`)
       c. Call `gitBranchCreate(repoPath, branchName)` via dynamic import from `@/lib/tauri`
       d. Call `gitBranchSwitch(repoPath, branchName)` via dynamic import
       e. Set currentStep='implementing'
       f. Call `runPilotImplement(issueId, repoPath, onOutput)` where onOutput appends to `this.output` array (cap at 500 lines via shift). Store the returned sidecarId from the SidecarOutput events.
       g. Check exit_code: if non-zero, set currentStep='error', error=`pilot implement failed with exit code ${code}`, return
       h. Set currentStep='staging'
       i. Call `gitStatus(repoPath)`, collect all file paths (staged and unstaged), call `gitStage(repoPath, allPaths)` via dynamic import
       j. Set currentStep='committing'
       k. Call `gitCommit(repoPath, `feat: implement ${issueId}`)` — store returned OID in commitOid
       l. Set currentStep='pushing'
       m. Call `gitPush(repoPath, () => {})` via dynamic import (progress not needed for this)
       n. Set currentStep='done', isRunning=false
       o. Each step wrapped in try/catch — on error: set currentStep='error', error=message, isRunning=false
       p. After completion (success or failure), emit a CustomEvent 'implement-complete' on window with detail: { issueId, success: currentStep === 'done', error }
     - `cancel(): Promise<void>` — if sidecarId is set, call cancelSidecar(sidecarId), set currentStep='error', error='Cancelled by user', isRunning=false
     - `reset(): void` — clear all fields to defaults
   - Use `runInAction` for all state mutations after async operations (same pattern as GitStore)
   - All tauri imports via dynamic `await import('@/lib/tauri')` pattern (same as GitStore)

   - Add `this.implement.reset()` to `reset()` method
   - Add `useImplementStore()` hook: `return useStores().implement`
   - Export `useImplementStore` from RootStore
