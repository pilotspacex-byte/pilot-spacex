Create 3 observer components + barrel export in `frontend/src/features/git/`.

**1. `git-status-panel.tsx` — GitStatusPanel**

An observer component showing file status and pull/push actions for the active repo.

Props: `{ repoPath: string }` — when mounted, calls `gitStore.setRepoPath(repoPath)`.

Layout (vertical stack):
- **Header row:** Current branch name (from `gitStore.currentBranch`), ahead/behind badges (e.g., "2 ahead", "3 behind" — only shown if non-zero), refresh button (calls `gitStore.refreshAll()`)
- **Pull/Push buttons row:**
  - "Pull" Button (variant="outline", size="sm"): onClick calls `gitStore.pull()`. While `isPulling`, show Progress bar (shadcn `Progress` component) bound to `pullProgress?.pct` with message text, and disable Push button. Show `pullError` as red text below if set.
  - "Push" Button (variant="outline", size="sm"): onClick calls `gitStore.push()`. While `isPushing`, show Progress bar bound to `pushProgress?.pct`. Show `pushError` as red text below if set.
- **File list section:**
  - Section heading "Changes" with count badge
  - Grouped by: Staged (green dot), Modified (yellow dot), Untracked (gray dot)
  - Each file: status icon + relative path in mono text + staged badge
  - If no files: "Working tree clean" message in muted text
  - Show `gitStore.error` as red text if set

Icons: use lucide-react — `ArrowDown` for pull, `ArrowUp` for push, `RefreshCw` for refresh, `GitBranch` for branch name. Status dots: colored `<span>` with `rounded-full size-2`.

Use shadcn/ui: `Button`, `Badge`, `Progress`, `ScrollArea` (for file list if > 10 files), `Separator`.

**2. `branch-selector.tsx` — BranchSelector**

An observer component providing a dropdown for branch operations.

Props: `{ repoPath: string }` — does NOT call setRepoPath (assumes GitStatusPanel already set it; or parent sets it).

Layout:
- **Trigger:** A Button (variant="outline", size="sm") showing current branch name + ChevronDown icon. Uses shadcn `Popover` + `Command` (cmdk) pattern for searchable dropdown.
- **Popover content:**
  - Search input for filtering branches
  - **Local branches section:** List of local branches from `gitStore.branches.filter(b => !b.is_remote)`. Current branch has a check mark. Click to switch (`gitStore.switchBranch(name)`). Each has a trash icon button for delete (`gitStore.deleteBranch(name)`) — hidden for current branch.
  - **Remote branches section:** Collapsible group of remote branches, display-only (no switch action for remote-only branches).
  - **Create branch footer:** Input + "Create" button. Calls `gitStore.createBranch(name)`, clears input, closes popover.
- Error display: show `gitStore.error` inline if branch operation fails.

Use shadcn/ui: `Popover`, `PopoverTrigger`, `PopoverContent`, `Command`, `CommandInput`, `CommandList`, `CommandEmpty`, `CommandGroup`, `CommandItem`, `Button`, `Separator`.
Icons: `GitBranch`, `Check`, `Trash2`, `ChevronDown`, `Plus` from lucide-react.

**3. `conflict-banner.tsx` — ConflictBanner**

An observer component that renders ONLY when `gitStore.hasConflicts` is true.

Layout: A prominent warning banner (yellow/amber background) at the top of the git panel:
- Icon: `AlertTriangle` from lucide-react
- Title: "Merge conflicts detected"
- Description: "{n} file(s) have conflicts that need to be resolved manually"
- Expandable file list: each conflicted file path in mono text
- "Dismiss" button: calls `gitStore.dismissConflicts()`

Use Tailwind: `bg-amber-50 dark:bg-amber-950/20 border-amber-200 dark:border-amber-800` for the banner. Text: `text-amber-800 dark:text-amber-200`.
