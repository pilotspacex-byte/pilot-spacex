Add two new public async commands to git.rs: `git_pull` and `git_push`. Both follow the established `spawn_blocking` + `Channel<GitProgress>` pattern from `git_clone`.

**`git_pull(repo_path: String, on_progress: Channel<GitProgress>) -> Result<GitPullResult, String>`**

New return struct:
```rust
#[derive(serde::Serialize, Clone)]
pub struct GitPullResult {
    pub updated: bool,
    pub conflicts: Vec<String>,  // file paths relative to repo root
}
```

Implementation:
1. Open repo: `Repository::open(&repo_path)` inside spawn_blocking
2. Get remote: `repo.find_remote("origin")` — error if no origin
3. Build RemoteCallbacks with:
   - `transfer_progress` callback: same 2% throttle pattern (Cell<u32>) sending to Channel
   - `credentials` callback: identical to git_clone's (keyring PAT, 3-attempt loop detection)
4. Fetch: `remote.fetch(&["refs/heads/*:refs/remotes/origin/*"], Some(&mut fo), None)` with FetchOptions containing the callbacks
5. After fetch, send progress 50% "Fetch complete, merging..."
6. Get the current branch name: `repo.head()?.shorthand()` — error if HEAD is detached
7. Find merge target: `repo.find_reference(&format!("refs/remotes/origin/{}", branch_name))` — the upstream tracking ref
8. Get annotated commit: `repo.reference_to_annotated_commit(&fetch_head_ref)`
9. Merge analysis: `repo.merge_analysis(&[&annotated_commit])`
   - If `analysis.is_up_to_date()` → return `GitPullResult { updated: false, conflicts: vec![] }`
   - If `analysis.is_fast_forward()` → fast-forward: `reference.set_target(annotated_commit.id(), "fast-forward pull")`, then `repo.checkout_head(Some(CheckoutBuilder::default().force()))`
   - If `analysis.is_normal()` → do `repo.merge(&[&annotated_commit], None, None)` then check status for conflicts
10. After merge, check for conflicts: `repo.statuses(None)` and filter entries with `status.is_conflicted()` → collect file paths into `conflicts: Vec<String>`
11. If conflicts is non-empty, do NOT auto-commit — return the conflict list so the user can resolve manually
12. If no conflicts and merge was performed (normal merge), auto-commit the merge: create merge commit with both parent HEADs
13. Send progress 100% "Pull complete"
14. Return `GitPullResult { updated: true, conflicts }`

**`git_push(repo_path: String, on_progress: Channel<GitProgress>) -> Result<(), String>`**

Implementation:
1. Open repo inside spawn_blocking
2. Get HEAD branch name — error if detached
3. Find remote "origin"
4. Build RemoteCallbacks with:
   - `push_transfer_progress` callback: throttled Channel progress (total/current objects)
   - `credentials` callback: identical to git_clone's pattern
5. Build PushOptions with the callbacks
6. Push: `remote.push(&[&format!("refs/heads/{}:refs/heads/{}", branch, branch)], Some(&mut push_opts))`
7. Send 100% "Push complete"
8. Return Ok(())

**Important implementation notes:**
- Both commands take `repo_path: String` (not `app: tauri::AppHandle`) — they operate on an already-cloned repo
- Reuse the exact same credential callback closure pattern from `git_clone` (keyring PAT, Cell<u32> attempt counter, max 3)
- Extract the credential callback into a helper function `build_credential_callback()` returning an `impl FnMut(...)` to avoid duplicating the closure in 3 places (clone, pull, push). If lifetime issues make this impractical, duplicate the closure inline.
- All git2 operations MUST be in `spawn_blocking` — `Repository` is not Send

**New structs:**

```rust
#[derive(serde::Serialize, Clone)]
pub struct FileStatus {
    pub path: String,
    pub status: String,  // "modified", "added", "deleted", "renamed", "untracked", "conflicted"
    pub staged: bool,
}

#[derive(serde::Serialize, Clone)]
pub struct GitRepoStatus {
    pub files: Vec<FileStatus>,
    pub branch: String,        // current branch name, or "HEAD (detached)"
    pub ahead: u32,            // commits ahead of upstream
    pub behind: u32,           // commits behind upstream
}

#[derive(serde::Serialize, Clone)]
pub struct BranchInfo {
    pub name: String,
    pub is_current: bool,
    pub is_remote: bool,
    pub upstream: Option<String>,  // e.g., "origin/main"
}
```

**`git_status(repo_path: String) -> Result<GitRepoStatus, String>`**

1. Open repo inside spawn_blocking
2. Get current branch: `repo.head()?.shorthand()` — use "HEAD (detached)" if detached
3. Compute ahead/behind: find upstream tracking branch via `repo.branch_upstream_name()`, then `repo.graph_ahead_behind(local_oid, upstream_oid)` — default to (0, 0) if no upstream
4. Get file statuses: `repo.statuses(Some(StatusOptions::new().include_untracked(true).recurse_untracked_dirs(true)))`
5. Map each status entry to `FileStatus`:
   - `STATUS_WT_MODIFIED` or `STATUS_INDEX_MODIFIED` → "modified" (staged = INDEX variant)
   - `STATUS_WT_NEW` → "untracked" (staged = false)
   - `STATUS_INDEX_NEW` → "added" (staged = true)
   - `STATUS_WT_DELETED` or `STATUS_INDEX_DELETED` → "deleted" (staged = INDEX variant)
   - `STATUS_WT_RENAMED` or `STATUS_INDEX_RENAMED` → "renamed" (staged = INDEX variant)
   - `STATUS_CONFLICTED` → "conflicted" (staged = false)
   - Note: A file can appear twice — once for staged and once for working tree changes. Include both entries.
6. Return `GitRepoStatus { files, branch, ahead, behind }`

**`git_branch_list(repo_path: String) -> Result<Vec<BranchInfo>, String>`**

1. Open repo inside spawn_blocking
2. Get current branch name (HEAD shorthand)
3. Iterate `repo.branches(Some(BranchType::Local))`:
   - For each: name, is_current (matches HEAD), upstream name via `branch.upstream()?.name()`
4. Iterate `repo.branches(Some(BranchType::Remote))`:
   - For each: name with `is_remote: true`
5. Return sorted by: current first, then local alpha, then remote alpha

**`git_branch_create(repo_path: String, name: String) -> Result<(), String>`**

1. Open repo inside spawn_blocking
2. Get HEAD commit: `repo.head()?.peel_to_commit()`
3. Create branch: `repo.branch(&name, &commit, false)` — `false` = don't force (error if exists)

**`git_branch_switch(repo_path: String, name: String) -> Result<(), String>`**

1. Open repo inside spawn_blocking
2. Set HEAD: `repo.set_head(&format!("refs/heads/{}", name))`
3. Checkout: `repo.checkout_head(Some(CheckoutBuilder::default().safe()))` — safe checkout refuses if uncommitted changes would be overwritten; return descriptive error

**`git_branch_delete(repo_path: String, name: String) -> Result<(), String>`**

1. Open repo inside spawn_blocking
2. Check current branch — refuse with error "Cannot delete the currently checked out branch" if `name` matches HEAD shorthand
3. Find branch: `repo.find_branch(&name, BranchType::Local)`
4. Delete: `branch.delete()` — git2 will error if unmerged; propagate this as a user-facing error message
