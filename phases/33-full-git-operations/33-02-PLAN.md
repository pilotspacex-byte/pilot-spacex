**New interfaces (add after existing GitCredentialInfo):**

```typescript
export interface GitPullResult {
  updated: boolean;
  conflicts: string[];
}

export interface FileStatus {
  path: string;
  status: 'modified' | 'added' | 'deleted' | 'renamed' | 'untracked' | 'conflicted';
  staged: boolean;
}

export interface GitRepoStatus {
  files: FileStatus[];
  branch: string;
  ahead: number;
  behind: number;
}

export interface BranchInfo {
  name: string;
  is_current: boolean;
  is_remote: boolean;
  upstream: string | null;
}
```

**New wrapper functions:**

1. `gitPull(repoPath: string, onProgress: (p: GitProgress) => void): Promise<GitPullResult>` — same Channel pattern as gitClone. Creates `Channel<GitProgress>`, wires onmessage, invokes `git_pull` with `{ repoPath, onProgress: channel }`. Returns GitPullResult.

2. `gitPush(repoPath: string, onProgress: (p: GitProgress) => void): Promise<void>` — same Channel pattern. Invokes `git_push` with `{ repoPath, onProgress: channel }`.

3. `gitStatus(repoPath: string): Promise<GitRepoStatus>` — simple invoke, no Channel. Invokes `git_status` with `{ repoPath }`.

4. `gitBranchList(repoPath: string): Promise<BranchInfo[]>` — invokes `git_branch_list` with `{ repoPath }`.

5. `gitBranchCreate(repoPath: string, name: string): Promise<void>` — invokes `git_branch_create` with `{ repoPath, name }`.

6. `gitBranchSwitch(repoPath: string, name: string): Promise<void>` — invokes `git_branch_switch` with `{ repoPath, name }`.

7. `gitBranchDelete(repoPath: string, name: string): Promise<void>` — invokes `git_branch_delete` with `{ repoPath, name }`.

All follow the established pattern: `if (!isTauri()) throw new Error('Not in Tauri mode');` for mutation commands, `if (!isTauri()) return <default>;` for queries (gitStatus returns `{ files: [], branch: '', ahead: 0, behind: 0 }`; gitBranchList returns `[]`).

Add JSDoc comments to each wrapper matching the style of existing wrappers.

```typescript
'use client';

import { makeAutoObservable, runInAction } from 'mobx';
import type { GitProgress, GitPullResult, GitRepoStatus, BranchInfo } from '@/lib/tauri';
```

**Observable state:**

| Observable | Type | Default | Purpose |
|------------|------|---------|---------|
| `repoPath` | `string` | `''` | Active repo path (set by UI when user selects a project) |
| `status` | `GitRepoStatus \| null` | `null` | Current file statuses + branch + ahead/behind |
| `branches` | `BranchInfo[]` | `[]` | All local + remote branches |
| `isLoadingStatus` | `boolean` | `false` | Status fetch in progress |
| `isLoadingBranches` | `boolean` | `false` | Branch list fetch in progress |
| `isPulling` | `boolean` | `false` | Pull operation in progress |
| `pullProgress` | `GitProgress \| null` | `null` | Current pull progress |
| `pullResult` | `GitPullResult \| null` | `null` | Last pull result (updated flag + conflicts) |
| `pullError` | `string \| null` | `null` | Pull error message |
| `isPushing` | `boolean` | `false` | Push operation in progress |
| `pushProgress` | `GitProgress \| null` | `null` | Current push progress |
| `pushError` | `string \| null` | `null` | Push error message |
| `error` | `string \| null` | `null` | General error (status/branch operations) |

**Computed:**

| Computed | Returns | Logic |
|----------|---------|-------|
| `currentBranch` | `string` | `this.status?.branch ?? ''` |
| `hasConflicts` | `boolean` | `(this.pullResult?.conflicts.length ?? 0) > 0` |
| `conflictedFiles` | `string[]` | `this.pullResult?.conflicts ?? []` |
| `modifiedFiles` | `FileStatus[]` | `this.status?.files.filter(f => f.status !== 'untracked') ?? []` |
| `untrackedFiles` | `FileStatus[]` | `this.status?.files.filter(f => f.status === 'untracked') ?? []` |
| `stagedFiles` | `FileStatus[]` | `this.status?.files.filter(f => f.staged) ?? []` |

**Actions:**

1. `setRepoPath(path: string)` — sets repoPath, clears all state, calls `refreshAll()` if path non-empty
2. `async refreshStatus()` — calls `gitStatus(this.repoPath)`, updates `status`, handles errors
3. `async refreshBranches()` — calls `gitBranchList(this.repoPath)`, updates `branches`, handles errors
4. `async refreshAll()` — calls `refreshStatus()` and `refreshBranches()` in parallel via `Promise.all`
5. `async pull()` — sets isPulling, calls `gitPull(this.repoPath, progress_cb)`, updates pullProgress on each tick via `runInAction`, sets pullResult, clears isPulling, calls `refreshAll()` after complete
6. `async push()` — sets isPushing, calls `gitPush(this.repoPath, progress_cb)`, updates pushProgress, clears isPushing, calls `refreshAll()` after complete
7. `async createBranch(name: string)` — calls `gitBranchCreate`, then `refreshBranches()`
8. `async switchBranch(name: string)` — calls `gitBranchSwitch`, then `refreshAll()` (status changes on switch)
9. `async deleteBranch(name: string)` — calls `gitBranchDelete`, then `refreshBranches()`
10. `dismissConflicts()` — sets `this.pullResult = null` (user acknowledges conflicts)
11. `reset()` — resets all observables to defaults

All async actions: lazy `await import('@/lib/tauri')` for each IPC call. Use try/catch with `runInAction` for error state. Same pattern as ProjectStore.
