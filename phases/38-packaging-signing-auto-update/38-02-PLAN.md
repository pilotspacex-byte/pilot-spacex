   "windows": {

     }

   - `webviewInstallMode.downloadBootstrapper`: ensures WebView2 is installed on Windows if missing (minimal installer size)

Update `.github/workflows/tauri-build.yml`:

1. **Add Windows EV signing env vars** to the "Build Tauri app" step. Append to the existing env block (which already has APPLE_* vars from Plan 01):
   ```yaml
   # Windows EV signing via Azure Key Vault (set in GitHub repo secrets)
   AZURE_KEY_VAULT_URI: ${{ secrets.AZURE_KEY_VAULT_URI }}
   AZURE_KEY_VAULT_CLIENT_ID: ${{ secrets.AZURE_KEY_VAULT_CLIENT_ID }}

   AZURE_KEY_VAULT_TENANT_ID: ${{ secrets.AZURE_KEY_VAULT_TENANT_ID }}
   AZURE_KEY_VAULT_CERT_NAME: ${{ secrets.AZURE_KEY_VAULT_CERT_NAME }}
   ```
   Note: Tauri v2 tauri-action reads Azure Key Vault env vars for Windows signing. When secrets are empty/unset, it produces unsigned builds.

2. **Add Azure SignTool installation step** for Windows runner, placed BEFORE tauri-action:
   ```yaml
   - name: Install Azure SignTool (Windows)
     if: runner.os == 'Windows'
     run: dotnet tool install --global AzureSignTool
   ```
   This is required for tauri-action to sign with Azure Key Vault certificates on Windows.

3. **Verify Linux system dependencies are already correct.**
   The existing step installs: libwebkit2gtk-4.1-dev, libappindicator3-dev, librsvg2-dev, patchelf.
   These are sufficient for .deb and .AppImage production. No changes needed for the Linux runner.

4. **Add sidecar signing step for Windows** (conditional, like macOS sidecar signing in Plan 01):
   ```yaml
   - name: Sign sidecar binary (Windows)
     if: runner.os == 'Windows' && env.AZURE_KEY_VAULT_URI != ''
     env:
       AZURE_KEY_VAULT_URI: ${{ secrets.AZURE_KEY_VAULT_URI }}
       AZURE_KEY_VAULT_CLIENT_ID: ${{ secrets.AZURE_KEY_VAULT_CLIENT_ID }}
       AZURE_KEY_VAULT_CLIENT_SECRET: ${{ secrets.AZURE_KEY_VAULT_CLIENT_SECRET }}
       AZURE_KEY_VAULT_TENANT_ID: ${{ secrets.AZURE_KEY_VAULT_TENANT_ID }}
       AZURE_KEY_VAULT_CERT_NAME: ${{ secrets.AZURE_KEY_VAULT_CERT_NAME }}
     shell: pwsh
     run: |
       $sidecar = "tauri-app\src-tauri\binaries\pilot-cli-x86_64-pc-windows-msvc.exe"
       if (Test-Path $sidecar) {
         AzureSignTool sign -kvu $env:AZURE_KEY_VAULT_URI `
           -kvi $env:AZURE_KEY_VAULT_CLIENT_ID `
           -kvs $env:AZURE_KEY_VAULT_CLIENT_SECRET `
           -kvt $env:AZURE_KEY_VAULT_TENANT_ID `
           -kvc $env:AZURE_KEY_VAULT_CERT_NAME `
           -tr "http://timestamp.digicert.com" `
           -td sha256 `
           $sidecar
         Write-Host "Signed sidecar: $sidecar"
       } else {
         Write-Host "No sidecar binary found — skipping signing"
       }
   ```

5. **Update artifact upload paths** to be more specific about installer files:
   Keep the existing artifact upload step but ensure the path pattern captures .dmg, .deb, .AppImage, and .msi files from the bundle directory.
   The existing path `tauri-app/src-tauri/target/${{ matrix.target }}/release/bundle/` already captures everything in the bundle directory. This is correct — no change needed.
