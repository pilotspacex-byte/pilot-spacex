2. **Add sidecar download step** BEFORE the "Build Tauri app" step for macOS runners only:
   - Use `actions/download-artifact@v4` to download the sidecar binary for the current target triple
   - This requires the pilot-cli-build workflow to run first. Add a `needs` dependency or use `workflow_call`.
   - ALTERNATIVE (simpler): Add a separate job `build-sidecar` at the top of tauri-build.yml that builds the sidecar inline, OR download from the pilot-cli-build workflow's latest artifacts.
   - SIMPLEST approach (use this): Add a step on macOS runners that creates a stub sidecar binary if the real one doesn't exist. The real sidecar binary will be provided by the release workflow (Plan 03). For CI builds (non-release), unsigned sidecar stubs are acceptable since the build is for testing.
   - Actually, the CORRECT approach for signing: Add a step on macOS runners BEFORE tauri-action that:
     a. Downloads the sidecar artifact from the pilot-cli-build workflow (use `actions/github-script` or `gh` CLI to fetch the latest artifact)
     b. Signs the sidecar binary with the Apple Developer ID certificate using `codesign --force --options runtime --sign "$APPLE_SIGNING_IDENTITY" --entitlements src-tauri/Entitlements.plist`
     c. This must happen BEFORE tauri-action bundles the app

   For this plan, use a conditional step: if APPLE_SIGNING_IDENTITY is set (release build), sign the sidecar. Otherwise skip (CI test build).

3. **Uncomment and wire Apple signing secrets** in the "Build Tauri app" step's env block.
   Change the commented-out lines to active env vars:
   ```yaml
   env:
     GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
     NEXT_TAURI: 'true'
     NEXT_PUBLIC_API_URL: 'https://api.pilotspace.io/api/v1'
     # macOS signing (set in GitHub repo secrets)
     APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
     APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
     APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
     APPLE_ID: ${{ secrets.APPLE_ID }}
     APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
     APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
   ```
   Note: tauri-action reads these env vars automatically — no additional args needed.
   When secrets are empty/unset, tauri-action produces unsigned builds (backward compatible).

4. **Add sidecar signing step** for macOS runners, placed AFTER frontend install but BEFORE tauri-action:
   ```yaml
   - name: Sign sidecar binary (macOS)
     if: runner.os == 'macOS' && env.APPLE_SIGNING_IDENTITY != ''
     env:
       APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
       TARGET: ${{ matrix.target }}
     run: |
       SIDECAR="tauri-app/src-tauri/binaries/pilot-cli-${TARGET}"
       if [ -f "$SIDECAR" ]; then
         codesign --force --options runtime \
           --sign "$APPLE_SIGNING_IDENTITY" \
           --entitlements tauri-app/src-tauri/Entitlements.plist \
           "$SIDECAR"
         echo "Signed sidecar: $SIDECAR"
         codesign -dv "$SIDECAR"
       else
         echo "No sidecar binary found at $SIDECAR — skipping signing"
       fi
   ```

5. **Keep existing Linux and Windows runners unchanged** — Plan 02 handles those.

6. **Add trigger for tags** to support release builds:
   Add `tags: ['v*']` under `push.branches` so that creating a version tag triggers a build:
   ```yaml
   on:
     push:
       branches: [main, develop, 'feat/tauri-*']
       tags: ['v*']
   ```

IMPORTANT: Do NOT modify the Linux or Windows runner steps — Plan 02 will handle those.
IMPORTANT: tauri-action automatically handles notarization when APPLE_ID + APPLE_PASSWORD + APPLE_TEAM_ID are set. No manual `xcrun notarytool submit` needed.
