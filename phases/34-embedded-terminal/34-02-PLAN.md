     - `setActiveSession(id: string | null)` — set the active session ID
     - `setPanelHeight(height: number)` — store the panel height
     - `reset()` — reset all fields to defaults
   - Use `makeAutoObservable(this)` in constructor

5. **Create `frontend/src/features/terminal/hooks/useTerminal.ts`:**
   - Custom hook managing the xterm.js Terminal instance and PTY session lifecycle
   - Parameters: `containerRef: React.RefObject<HTMLDivElement | null>`
   - Returns: `{ isReady: boolean; sessionId: string | null }`

   **Implementation:**
   ```typescript
   'use client';

   import { useEffect, useRef, useCallback, useState } from 'react';
   import type { Terminal } from '@xterm/xterm';
   import type { FitAddon } from '@xterm/addon-fit';
   ```

   **useEffect lifecycle (on mount when container is available):**
   a. Dynamically import xterm: `const { Terminal } = await import('@xterm/xterm');`
   b. Dynamically import fit addon: `const { FitAddon } = await import('@xterm/addon-fit');`
   c. Import xterm CSS: `await import('@xterm/xterm/css/xterm.css');`
   d. Create Terminal instance:
      ```typescript
      const term = new Terminal({
        scrollback: 10_000,
        fontSize: 13,
        fontFamily: "'JetBrains Mono', 'Fira Code', 'Cascadia Code', 'Menlo', monospace",
        theme: {
          background: '#1a1b26',
          foreground: '#c0caf5',
          cursor: '#c0caf5',
          selectionBackground: '#33467c',
        },
        cursorBlink: true,
        allowProposedApi: true,
      });
      ```
   e. Create and load FitAddon: `const fitAddon = new FitAddon(); term.loadAddon(fitAddon);`
   f. Open terminal in container: `term.open(containerRef.current);`
   g. Call `fitAddon.fit()` to size terminal to container
   h. Get terminal dimensions: `const { rows, cols } = term;`
   i. Create PTY session via IPC:
      ```typescript
      const { createTerminal, writeTerminal } = await import('@/lib/tauri');
      const session = await createTerminal(rows, cols, (output) => {
        term.write(output.data);
      });
      ```
   j. Wire keyboard input to PTY:
      ```typescript
      term.onData((data: string) => {
        writeTerminal(session.session_id, data);
      });
      ```
   k. Wire resize events:
      ```typescript
      const resizeObserver = new ResizeObserver(() => {
        fitAddon.fit();
        const { resizeTerminal } = await import('@/lib/tauri');
        resizeTerminal(session.session_id, term.rows, term.cols);
      });
      resizeObserver.observe(containerRef.current);
      ```
      NOTE: Since ResizeObserver callback cannot be async, pre-import resizeTerminal before setting up the observer, or use a sync wrapper that calls the async function without await (fire-and-forget for resize).

   **useEffect cleanup (on unmount):**
   a. Close PTY session: `closeTerminal(session.session_id)`
   b. Disconnect ResizeObserver
   c. Dispose terminal: `term.dispose()`
   d. Clear the active session in TerminalStore

   **Important patterns:**
   - All `@xterm/*` and `@/lib/tauri` imports MUST be dynamic (inside useEffect or callbacks) — not top-level
   - The `term.write(output.data)` call is already safe because the Rust side batches at 16ms
   - Use `useRef` to hold the Terminal instance and session ID between renders (NOT useState — avoids re-render on every output chunk)
   - Return `isReady` state (true after PTY session is created) and `sessionId`

1. **Create `frontend/src/features/terminal/components/TerminalPanel.tsx`:**
   - MobX `observer()` component that reads `terminalStore.isOpen` and renders the terminal panel conditionally
   - Layout: fixed bottom panel (like VS Code integrated terminal)

   **Structure:**
   ```tsx
   'use client';

   import { observer } from 'mobx-react-lite';
   import { useRef, useCallback } from 'react';
   import { useTerminalStore } from '@/stores/RootStore';
   import { useTerminal } from '../hooks/useTerminal';
   import { isTauri } from '@/lib/tauri';
   ```

   **Component:**
   - If `!isTauri()`, return null (terminal only available in desktop app)
   - If `!terminalStore.isOpen`, return null (panel hidden)
   - Render structure:
     ```
     <div className="terminal-panel"> (absolute/fixed bottom panel)
       <div className="terminal-header"> (toolbar with title + close button)
         <span>Terminal</span>
         <button onClick={terminalStore.close}>X</button>
       </div>
       <div ref={containerRef} className="terminal-container" /> (xterm mounts here)
     </div>
     ```

   **Styling (Tailwind classes):**
   - Outer panel: `fixed bottom-0 left-0 right-0 z-50 border-t border-border bg-[#1a1b26]` with height from `terminalStore.panelHeight`
   - Header bar: `flex items-center justify-between px-3 h-8 bg-muted/80 border-b border-border text-xs text-muted-foreground`
   - Terminal container: `flex-1 overflow-hidden` (xterm needs a container that has explicit dimensions)
   - The panel height should use inline style: `style={{ height: terminalStore.panelHeight }}`

   **Header bar contents:**
   - Left side: "Terminal" label with a terminal icon (use lucide-react `Terminal` icon if available, or plain text)
   - Right side: Close button (`X` icon) that calls `terminalStore.close()` — this hides the panel but does NOT destroy the PTY session (session persists across hide/show)

   **Keyboard shortcut:**
   - Add a global keyboard listener (in a useEffect) for `Ctrl+\`` (backtick) to toggle the terminal panel
   - Use `terminalStore.toggle()` on the shortcut
   - Prevent the shortcut from firing when the user is typing in an input/textarea (check `document.activeElement`)

   **useTerminal integration:**
   - Pass `containerRef` to `useTerminal(containerRef)`
   - The hook manages the full xterm lifecycle internally

   **Important:**
   - The TerminalPanel must be wrapped in `observer()` to react to `terminalStore.isOpen` changes
   - The container `div` must have a stable ref — do NOT conditionally render it based on `isReady` from the hook (xterm needs the DOM element to mount into)
   - xterm.css must be imported (the hook handles this dynamically)
