<objective>
Build the Rust PTY backend for the embedded terminal — create/write/resize/close PTY sessions with batched output streaming over Tauri IPC Channel.

Purpose: TERM-02 (full PTY support for interactive programs), TERM-03 (streaming output), TERM-04 (batched IPC to prevent memory leaks). The Rust side must batch output at 16ms intervals to avoid the confirmed Tauri IPC memory leak (issues #12724 and #13133) and provide explicit session lifecycle cleanup.

Output: `commands/terminal.rs` with 4 IPC commands, `TerminalSessions` managed state, typed TypeScript wrappers in `tauri.ts`.

2. **Create `commands/terminal.rs`:**

   **Types:**
   ```rust
   #[derive(serde::Serialize, Clone)]
   pub struct TerminalOutput {
       pub session_id: String,
       pub data: String,
   }

   #[derive(serde::Serialize, Clone)]
   pub struct TerminalSessionInfo {
       pub session_id: String,
   }
   ```

   **Managed State:**
   ```rust
   pub struct TerminalSessions {
       sessions: std::sync::Mutex<HashMap<String, TerminalSession>>,
   }
   ```

   Where `TerminalSession` holds:
   - The PTY master writer (for writing stdin)
   - A handle to the reader thread (for cleanup on close)
   - The child process handle (for kill on close)

   **Command: `create_terminal`**
   - Signature: `pub async fn create_terminal(state: tauri::State<'_, TerminalSessions>, on_output: Channel<TerminalOutput>, rows: u16, cols: u16) -> Result<TerminalSessionInfo, String>`
   - Generate a UUID v4 session ID
   - Detect the user's default shell:
     - macOS/Linux: read `$SHELL` env var, default to `/bin/bash`
     - Windows: use `cmd.exe` (or `powershell.exe` if `$COMSPEC` not found)
   - Create a PTY with the given rows/cols dimensions
   - Spawn the shell process in the PTY
   - **CRITICAL: Batched output reader thread:**
     - Spawn a `std::thread::spawn` (NOT tokio::spawn — PTY reads are blocking I/O)
     - In the thread, read from PTY master into a buffer
     - Accumulate output into a `Vec<u8>` buffer
     - Every 16ms, if the buffer is non-empty, convert to UTF-8 lossy String, send via `channel.send(TerminalOutput { session_id, data })`, and clear the buffer
     - Use `std::time::Instant` to track the 16ms interval — after each read, check if 16ms has elapsed since last flush; if so, flush the accumulated buffer
     - When PTY read returns 0 bytes (EOF) or error, flush any remaining buffer, then exit the thread
     - The thread must hold a clone of the Channel and session_id
   - Store the session in the `TerminalSessions` map
   - Return `TerminalSessionInfo { session_id }`

   **Command: `write_terminal`**
   - Signature: `pub async fn write_terminal(state: tauri::State<'_, TerminalSessions>, session_id: String, data: String) -> Result<(), String>`
   - Look up session by ID in the mutex-guarded HashMap
   - Write `data.as_bytes()` to the PTY master writer
   - Return error if session not found

   **Command: `resize_terminal`**
   - Signature: `pub async fn resize_terminal(state: tauri::State<'_, TerminalSessions>, session_id: String, rows: u16, cols: u16) -> Result<(), String>`
   - Look up session by ID
   - Call PTY resize method with new rows/cols (this sends SIGWINCH to the child process)
   - Return error if session not found

   **Command: `close_terminal`**
   - Signature: `pub async fn close_terminal(state: tauri::State<'_, TerminalSessions>, session_id: String) -> Result<(), String>`
   - Remove session from the HashMap
   - Kill the child process (best-effort — may already be exited)
   - The reader thread will naturally exit when the PTY master is dropped (read returns EOF)
   - Return Ok even if session was already closed (idempotent)

   **Important implementation notes:**
   - Use `Mutex<HashMap<String, TerminalSession>>` NOT `RwLock` — write operations are frequent (every keystroke goes through write_terminal)
   - The PTY master writer must be cloneable or wrapped in `Arc<Mutex<>>` so the reader thread and write command can coexist
   - On drop/close, dropping the PTY master will send EOF to the child, causing it to exit
   - Do NOT use Tauri event system (`emit_all`) — use `Channel` API only (Pitfall 7)

2. **Add `createTerminal` wrapper:**
   ```typescript
   /**
    * Create a new PTY terminal session.
    * Returns session info with a unique session_id.
    * Output is streamed via the onOutput callback (batched at 16ms by Rust).
    * @param rows - Initial terminal height in rows
    * @param cols - Initial terminal width in columns
    * @param onOutput - Callback invoked with batched terminal output chunks
    */
   export async function createTerminal(
     rows: number,
     cols: number,
     onOutput: (output: TerminalOutput) => void
   ): Promise<TerminalSessionInfo> {
     if (!isTauri()) throw new Error('Not in Tauri mode');
     const { invoke, Channel } = await import('@tauri-apps/api/core');
     const channel = new Channel<TerminalOutput>();
     channel.onmessage = onOutput;
     return invoke<TerminalSessionInfo>('create_terminal', {
       rows,
       cols,
       onOutput: channel,
     });
   }
   ```

3. **Add `writeTerminal` wrapper:**
   ```typescript
   /**
    * Write data (keystrokes) to a terminal session's stdin.
    * @param sessionId - The session ID returned by createTerminal
    * @param data - Raw character data to write to the PTY
    */
   export async function writeTerminal(
     sessionId: string,
     data: string
   ): Promise<void> {
     if (!isTauri()) return;
     const { invoke } = await import('@tauri-apps/api/core');
     await invoke('write_terminal', { sessionId, data });
   }
   ```

4. **Add `resizeTerminal` wrapper:**
   ```typescript
   /**
    * Resize a terminal session's PTY dimensions.
    * Sends SIGWINCH to the child process so programs like vim/htop re-render.
    * @param sessionId - The session ID returned by createTerminal
    * @param rows - New height in rows
    * @param cols - New width in columns
    */
   export async function resizeTerminal(
     sessionId: string,
     rows: number,
     cols: number
   ): Promise<void> {
     if (!isTauri()) return;
     const { invoke } = await import('@tauri-apps/api/core');
     await invoke('resize_terminal', { sessionId, rows, cols });
   }
   ```

5. **Add `closeTerminal` wrapper:**

     if (!isTauri()) return;
