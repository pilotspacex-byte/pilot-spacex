**Section 1: Project Directory**

Title: "Project Directory" with description: "Base directory where cloned repositories are stored."

- Display the current base path in a readonly `Input` field.
- "Change..." button next to the input that:
  1. Calls `openFolderDialog()` from `@/lib/tauri`.
  2. If user selects a folder (non-null), calls `setProjectsDir(selectedPath)`.
  3. Updates the displayed path.
- "Reset to Default" link/button that calls `setProjectsDir('')` (empty string signals reset to default `~/PilotSpace/projects/`), then reloads the path.
- Show the resolved path after save with a success indicator.

Implementation:
```typescript
'use client';

import { useState, useEffect, useCallback } from 'react';
```

On mount (`useEffect`), load current path via `getProjectsDir()` and current credentials via `getGitCredentials()`.

**Section 2: Git Credentials**

Title: "Git Credentials" with description: "HTTPS credentials for git operations (clone, push, pull)."

- Show current credential status:
  - If credentials exist: "Username: {username}" with a green check icon, "PAT: Configured" with a green check icon.
  - If no credentials: "No credentials configured" with a muted warning.
- Form fields:
  - `Input` for "Username" (e.g., GitHub username).
  - `Input` for "Personal Access Token" with type="password" (toggle visibility with eye icon).
- "Save Credentials" button that calls `setGitCredentials(username, pat)`.
- After save: refresh credential status display, clear the PAT input field.
- Show success/error feedback inline.

The PAT input should:
- Default to empty (never pre-populate with the actual PAT — it is not returned from Rust).
- Show placeholder "Enter new PAT to update" when credentials already exist.
- Show placeholder "Enter your Personal Access Token" when no credentials exist.

Use standard Tailwind spacing: `space-y-6` between sections, `space-y-3` within sections.
Use `Separator` between the two sections if available, otherwise a `border-t` divider.

Export: `export { DesktopSettingsPage }` (named export, matching the lazy-import pattern).
