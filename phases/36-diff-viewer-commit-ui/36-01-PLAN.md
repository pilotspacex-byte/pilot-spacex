**New struct:**
```rust
#[derive(serde::Serialize, Clone)]
pub struct FileDiff {
    pub path: String,
    /// The unified diff text (includes --- +++ @@ lines). Empty string if binary.
    pub diff: String,
    /// true if the file is binary (no text diff available)
    pub is_binary: bool,
}
```

**git_diff command** (DIFF-01):
- Signature: `pub async fn git_diff(repo_path: String, file_path: Option<String>) -> Result<Vec<FileDiff>, String>`
- Opens repo with `Repository::open`, runs in `spawn_blocking`
- If `file_path` is `Some(path)`: return diff for that single file. If `None`: return diffs for ALL changed files (working tree + index changes).
- Use `repo.diff_index_to_workdir(None, Some(&mut diff_opts))` for unstaged changes and `repo.diff_tree_to_index(head_tree, None, Some(&mut diff_opts))` for staged changes
- For each `DiffDelta`, check `delta.flags().is_binary()` — if binary, set `is_binary: true` and `diff: String::new()`
- For text diffs, use `diff.print(DiffFormat::Patch, |_delta, _hunk, line| { ... })` to accumulate the unified diff string. Collect the output per-file by tracking the current file path across delta callbacks.
- When `file_path` is Some, filter by `delta.new_file().path()` matching the requested path
- IMPORTANT: Import `DiffFormat`, `DiffOptions` from `git2`

**git_stage command** (DIFF-02):
- Signature: `pub async fn git_stage(repo_path: String, paths: Vec<String>) -> Result<(), String>`
- Opens repo, gets mutable index via `repo.index()`
- For each path: `index.add_path(Path::new(&path))` — handles both modified and new files
- After all paths: `index.write()` to persist the index to disk
- Run in `spawn_blocking`

**git_unstage command** (DIFF-02):
- Signature: `pub async fn git_unstage(repo_path: String, paths: Vec<String>) -> Result<(), String>`
- Opens repo, gets HEAD commit tree: `repo.head()?.peel_to_tree()?`
- For each path: `repo.reset_default(Some(&head_tree_obj), &[&path])` — this resets the index entry for that path back to HEAD without touching the working tree
- Use `repo.head()?.peel(ObjectType::Tree)?` to get the tree as a generic Object for reset_default
- Run in `spawn_blocking`
- IMPORTANT: Import `ObjectType` from git2

**git_commit command** (DIFF-03):
- Signature: `pub async fn git_commit(repo_path: String, message: String) -> Result<String, String>`
- Opens repo, gets index, writes tree from index: `index.write_tree()`
- Gets HEAD commit as parent: `repo.head()?.peel_to_commit()?`
- Gets signature: `repo.signature()` — falls back to `Signature::now("Pilot Space", "noreply@pilotspace.dev")` if no git config
- Creates commit: `repo.commit(Some("HEAD"), &sig, &sig, &message, &tree, &[&parent])`
- Returns the commit OID as a hex string: `oid.to_string()`
- Handle the case where the repo has no commits (initial commit): if `repo.head()` fails with `is_unborn_head()`, pass empty parents slice `&[]`
- Run in `spawn_blocking`
- IMPORTANT: Import `Signature` from git2
