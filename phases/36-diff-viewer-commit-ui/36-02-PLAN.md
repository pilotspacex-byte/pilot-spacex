Create `frontend/src/features/git/components/diff-viewer.tsx` — a component that renders unified diff text with colored line annotations. Do NOT use any external diff rendering library (react-diff-view, @git-diff-view/react) — they add unnecessary weight. Instead, parse the unified diff text directly since git2-rs already provides it in standard format.

**Component structure:**

```typescript

import { observer } from 'mobx-react-lite';

```

**DiffViewer props:**

```typescript
interface DiffViewerProps {
  /** Maximum height of the diff viewer. Defaults to '400px'. */
  maxHeight?: string;
}
```

**Diff line parsing** — create a helper function `parseDiffLines(diffText: string)` that splits the unified diff text into typed line objects:

```typescript
interface DiffLine {
  type: 'header' | 'hunk' | 'addition' | 'deletion' | 'context';
  content: string;
  oldLineNo: number | null;
  newLineNo: number | null;
}
```

Rules for parsing:
- Lines starting with `---` or `+++` → type: `'header'`
- Lines starting with `@@` → type: `'hunk'`, parse the `@@ -oldStart,oldCount +newStart,newCount @@` to reset line number counters
- Lines starting with `+` (but not `+++`) → type: `'addition'`, increment newLineNo only
- Lines starting with `-` (but not `---`) → type: `'deletion'`, increment oldLineNo only
- All other lines → type: `'context'`, increment both line numbers
- Track oldLineNo and newLineNo as running counters, reset on each hunk header

**Rendering:**

The component reads from `useGitStore()`:
- If `gitStore.selectedFilePath` is null → show empty state: "Select a file to view its diff"
- If `gitStore.isLoadingDiff` → show centered `<Loader2 className="animate-spin" />`
- If `gitStore.diffError` → show error in destructive text
- Find the matching `FileDiff` from `gitStore.fileDiffs` where `path === gitStore.selectedFilePath`
- If the FileDiff has `is_binary: true` → show `<Binary />` icon + "Binary file — no diff available"
- If the FileDiff diff is empty string and NOT binary → show "No changes"
- Otherwise, parse with `parseDiffLines()` and render

**Use `react-virtuoso` (already installed) for virtualization** (DIFF-05 performance):
- Wrap the diff lines in a `<Virtuoso>` component with `totalCount={lines.length}` and `style={{ height: maxHeight }}` — this prevents UI freeze on diffs with 1000+ lines
- Each line renders as a single row via `itemContent={(index) => <DiffLineRow line={lines[index]} />}`

**DiffLineRow sub-component** — renders a single diff line with:
- Fixed-width line number gutters (w-12 each for old/new, text-right, tabular-nums, text-muted-foreground text-xs font-mono)
- Content area with font-mono text-sm whitespace-pre and overflow-x-auto
- Background colors per type using Tailwind:
  - `addition`: `bg-green-500/10` background, `text-green-700 dark:text-green-400` for line numbers
  - `deletion`: `bg-red-500/10` background, `text-red-700 dark:text-red-400` for line numbers
  - `hunk`: `bg-blue-500/10` background, `text-blue-600 dark:text-blue-400` text, italic
  - `header`: `text-muted-foreground` italic
  - `context`: default (no special background)
- For `addition` lines, render content without the leading `+` character
- For `deletion` lines, render content without the leading `-` character
- For `context` lines, render content without the leading space character

**Memoize** the `parseDiffLines()` call with `useMemo` keyed on the diff text string.
