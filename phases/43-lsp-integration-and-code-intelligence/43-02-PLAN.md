       - Add a `handleDiagnosticNavigate` callback that receives (uri: string, line: number, column: number):
         a. Extract the file path from the uri (strip the `file:///` or `inmemory://model/` prefix to get the relative path).
         b. Check if the diagnostic's file matches the currently active file. Read the active file from the editor store (e.g., `fileStore.activeFile`).
         c. If the file is different from the active file: dispatch a `CustomEvent('editor:open-file', { detail: { path } })` or use the file store's `openFile` method to switch to the target file first, THEN dispatch the navigate event.
         d. Dispatch `window.dispatchEvent(new CustomEvent('symbol-outline:navigate', { detail: { uri, line } }))` to navigate the editor to the line. Include BOTH `uri` and `line` in the detail payload so the listener can verify the correct file is active.
         e. If cross-file open is not straightforward given the current store API, narrow the implementation to navigate within the active file only, but log a console.warn when a cross-file diagnostic is clicked: `console.warn('Cross-file diagnostic navigation not yet supported:', uri)`.
