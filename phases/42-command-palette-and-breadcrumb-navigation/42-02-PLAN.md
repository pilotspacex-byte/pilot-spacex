       ```typescript
       export type SymbolKind = 'heading' | 'pm-block' | 'function' | 'class' | 'variable' | 'interface' | 'method' | 'property';

       export interface DocumentSymbol {
         name: string;
         kind: SymbolKind;
         line: number;       // 1-based
         level: number;      // Heading level (1-6) or 0 for flat symbols
         children: DocumentSymbol[];
       }
       ```

    2. Create `markdownSymbols.ts`:
       - `parseMarkdownSymbols(content: string): DocumentSymbol[]`
       - Parse lines: `^(#{1,6})\s+(.+)$` for headings
       - Parse PM blocks: reuse `PM_BLOCK_REGEX` from `pmBlockMarkers.ts` -- extract type as name
       - Build hierarchy: H2 nests under preceding H1, H3 under preceding H2, etc.
       - PM blocks nest under the most recent heading at their level
       - Use a stack-based approach: maintain stack of (level, node) pairs; pop until finding a parent with lower level

    3. Create `useSymbolOutline.ts`:
       - Input: `content: string`, `language: string`, `editor: monaco.editor.IStandaloneCodeEditor | null`
       - For markdown/note files (language === 'markdown'): use `parseMarkdownSymbols(content)`
       - For code files: attempt Monaco OutlineModel API (dynamic import inside the hook):
         ```typescript
         const { OutlineModel } = await import('monaco-editor/esm/vs/editor/contrib/documentSymbols/browser/outlineModel.js');
         const { StandaloneServices } = await import('monaco-editor/esm/vs/editor/standalone/browser/standaloneServices.js');
         const { ILanguageFeaturesService } = await import('monaco-editor/esm/vs/editor/common/services/languageFeatures.js');
         ```
         Wrap in try/catch; fall back to empty array on failure.
       - Debounce: 500ms for markdown, 1000ms for code (use setTimeout)
       - Active symbol tracking: listen to `editor.onDidChangeCursorPosition`, find symbol whose line is <= cursorLine (deepest match)
       - Returns `{ symbols: DocumentSymbol[], activeSymbolId: string | null, isLoading: boolean }`
       - `activeSymbolId` is the `name` of the active symbol (for highlighting)
