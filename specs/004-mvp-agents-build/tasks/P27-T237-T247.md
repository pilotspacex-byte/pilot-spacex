# AI Stores

## Architecture

All AI features use MobX stores for state management with SSE streaming support.

## Store Hierarchy

```
AIStore (root)
├── GhostTextStore     - Real-time text suggestions
├── AIContextStore     - Issue context generation
├── PRReviewStore      - Pull request reviews
├── IssueExtractionStore - Note issue extraction
├── MarginAnnotationStore - Block annotations
├── ApprovalStore      - Approval queue management
├── AISettingsStore    - Workspace AI configuration
├── CostDashboardStore - Usage analytics
└── ConversationStore  - Multi-turn chat
```

## SSE Pattern

All streaming stores follow this pattern:

```typescript
class StreamingStore {
  private client: SSEClient | null = null;

  async startStream(): Promise<void> {
    this.abort(); // Cancel existing

    runInAction(() => {
      this.isLoading = true;
      this.error = null;
    });

    this.client = new SSEClient({
      url: apiUrl,
      onMessage: (event) => this.handleEvent(event),
      onComplete: () => runInAction(() => { this.isLoading = false; }),
      onError: (err) => runInAction(() => {
        this.isLoading = false;
        this.error = err.message;
      }),
    });

    await this.client.connect();
  }

  abort(): void {
    this.client?.abort();
    this.client = null;
    this.isLoading = false;
  }
}
```

## Usage

```typescript
const { ai } = useStore();

// Start streaming
await ai.aiContext.generateContext(issueId);

// Access reactive state
const isLoading = ai.aiContext.isLoading;
const phases = ai.aiContext.phases;
const result = ai.aiContext.result;

// Abort if needed
ai.aiContext.abort();
```
