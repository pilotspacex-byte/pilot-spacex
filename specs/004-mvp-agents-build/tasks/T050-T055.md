### Functionality

#### Primary Function

Perform comprehensive code review covering 5 aspects with severity classification. For large PRs, prioritize critical files (auth, security, API, models) and note skipped files.

#### Input/Output

| Direction | Type | Description | Constraints |
|-----------|------|-------------|-------------|
| Input | `PRReviewInput` | PR number, repo, project context | pr_number required |
| Output | `AsyncIterator[str]` | Streaming review content | Markdown format |

```python
@dataclass
class PRReviewInput:
    pr_number: int
    repo_full_name: str  # owner/repo
    project_id: str
    user_id: str
    workspace_id: str

@dataclass
class ReviewComment:
    file_path: str
    line_number: int
    severity: str  # critical, warning, info
    category: str  # architecture, security, quality, performance, documentation
    message: str
    suggestion: str | None = None

@dataclass
class PRReviewOutput:
    summary: str
    recommendation: str  # APPROVE, REQUEST_CHANGES, COMMENT
    comments: list[ReviewComment]
    critical_count: int
    warning_count: int
    info_count: int
    files_reviewed: int
    files_skipped: int
    cost_usd: float
```

#### Error Handling

| Error Type | Behavior | Recovery | Logging |
|------------|----------|----------|---------|
| PR not found | Return error | None | ERROR: pr_not_found |
| Large PR | Partial review | Note skipped files | WARNING: partial_review |
| Rate limit | Retry with backoff | Queue for later | WARNING: rate_limited |
| Tool error | Skip tool | Continue with available data | WARNING: tool_error |

### Implementation Guidelines

#### Structure

```python
from claude_agent_sdk import query, ClaudeAgentOptions
from claude_agent_sdk.types import AssistantMessage, TextBlock, ResultMessage
from pilot_space.ai.agents.sdk_base import SDKBaseAgent

class PRReviewAgent(SDKBaseAgent):
    """Unified AI PR Review covering 5 aspects (DD-006).

    Uses query() for one-shot comprehensive analysis.
    """

    AGENT_NAME = "pr_review"
    DEFAULT_MODEL = "claude-opus-4-5-20251101"
    MAX_BUDGET_USD = 20.0
    MAX_TURNS = 15

    # Large PR thresholds
    MAX_LINES_FULL_REVIEW = 5000
    MAX_FILES_FULL_REVIEW = 50

    SYSTEM_PROMPT = """You are an expert code reviewer analyzing pull requests.

Your review must cover ALL five aspects:

1. **🏗️ Architecture**: Layer boundaries, design patterns, dependency direction, SOLID principles
2. **🔒 Security**: OWASP vulnerabilities, secrets exposure, auth/authz issues, input validation
3. **✨ Code Quality**: Complexity, duplication, naming conventions, error handling, test coverage
4. **⚡ Performance**: N+1 queries, blocking I/O calls, resource leaks, algorithmic complexity
5. **📚 Documentation**: Missing docstrings, outdated comments, API documentation

Format your review as Markdown with severity indicators:
- 🔴 **Critical**: Must fix before merge (security vulnerabilities, data loss risk, breaking changes)
- 🟡 **Warning**: Should fix, but not blocking (code smells, minor issues)
- 🔵 **Info**: Suggestions for improvement (style, best practices)

For each issue, provide:
- **Location**: file:line
- **Description**: What's wrong
- **Suggestion**: How to fix it

End with overall recommendation: **APPROVE**, **REQUEST_CHANGES**, or **COMMENT**."""

    MCP_TOOLS = [
        "mcp__pilot_space__get_pr_details",
        "mcp__pilot_space__get_pr_diff",
        "mcp__pilot_space__search_codebase",
        "mcp__pilot_space__get_project_context",
    ]

    async def execute(
        self,
        input_data: PRReviewInput,
        user_id: str,
        workspace_id: str,
    ) -> AsyncIterator[str]:
        """Execute PR review with streaming output."""
        prompt = f"""Review Pull Request #{input_data.pr_number} in {input_data.repo_full_name}.

Use the available tools to:
1. Get PR details and metadata
2. Get the complete diff
3. Search the codebase for related patterns and conventions
4. Get project context for coding standards

Provide a comprehensive review covering ALL five aspects:
- 🏗️ Architecture
- 🔒 Security
- ✨ Code Quality
- ⚡ Performance
- 📚 Documentation

Be thorough but constructive. Praise good patterns when you see them."""

        options = self._build_options(
            system_prompt=self.SYSTEM_PROMPT,
            allowed_tools=self.MCP_TOOLS,
            model=self.DEFAULT_MODEL,
            max_budget_usd=self.MAX_BUDGET_USD,
        )

        async for message in query(prompt=prompt, options=options):
            if isinstance(message, AssistantMessage):
                for block in message.content:
                    if isinstance(block, TextBlock):
                        yield block.text

            if isinstance(message, ResultMessage):
                await self._track_result(message, user_id, workspace_id)
                yield f"\n\n---\n*Review Cost: ${message.total_cost_usd:.4f} | Duration: {message.duration_ms}ms*"
```

#### Large PR Handling

```python
PRIORITY_FILE_PATTERNS = [
    "auth/", "security/", "api/", "models/",
    "routes/", "routers/", "middleware/", "services/", "domain/"
]

def _filter_priority_files(self, changed_files: list[str]) -> tuple[list[str], list[str]]:
    """Separate priority files from others."""
    priority = []
    other = []
    for f in changed_files:
        if any(pattern in f for pattern in self.PRIORITY_FILE_PATTERNS):
            priority.append(f)
        else:
            other.append(f)
    return priority, other
```

Implements DD-006: Unified 5-aspect review system.

PR_REVIEW_SYSTEM_PROMPT = """You are an expert code reviewer analyzing pull requests.

## Review Aspects (Required)

You MUST cover ALL five aspects in your review:

### 1. 🏗️ Architecture
- Layer boundaries (presentation → application → domain → infrastructure)
- Design patterns usage and appropriateness
- Dependency direction (dependencies point inward)
- SOLID principles compliance
- Module cohesion and coupling

### 2. 🔒 Security
- OWASP Top 10 vulnerabilities (injection, XSS, CSRF, etc.)
- Secrets or credentials in code
- Authentication and authorization issues
- Input validation and sanitization
- Secure data handling (encryption, PII)

### 3. ✨ Code Quality

- Code duplication
- Naming conventions (clear, descriptive)
- Error handling patterns
- Test coverage for changes

### 4. ⚡ Performance
- N+1 database queries
- Blocking I/O in async code
- Resource leaks (connections, files, memory)
- Algorithmic complexity (unnecessary O(n²))
- Caching opportunities

### 5. 📚 Documentation
- Missing docstrings on public APIs
- Outdated or misleading comments
- API documentation (OpenAPI, schemas)
- README updates if needed
- Type annotations

## Severity Levels

- 🔴 **Critical**: Must fix before merge
  - Security vulnerabilities
  - Data loss or corruption risk
  - Breaking changes without migration
  - Production stability issues

- 🟡 **Warning**: Should fix, not blocking
  - Code smells
  - Minor bugs unlikely to cause issues
  - Missing tests for edge cases
  - Performance issues in non-critical paths

- 🔵 **Info**: Suggestions for improvement
  - Style improvements
  - Refactoring opportunities
  - Best practice suggestions
  - Documentation enhancements

## Output Format

For each issue found:
```markdown
### 🔴/🟡/🔵 [Category] Issue Title

**Location**: `file/path.py:123`

**Description**: What's wrong and why it matters.

**Suggestion**: How to fix it with code example if helpful.
```

End with:
```markdown
## Summary

- Critical: X issues
- Warnings: Y issues
- Info: Z suggestions

**Recommendation**: APPROVE / REQUEST_CHANGES / COMMENT

[Brief explanation of recommendation]
```
"""

def build_pr_review_prompt(
    pr_number: int,
    repo_full_name: str,
    partial_review: bool = False,
    files_to_review: list[str] | None = None,
    files_skipped: list[str] | None = None,
) -> str:
    """Build prompt for PR review."""
    base_prompt = f"""Review Pull Request #{pr_number} in {repo_full_name}.

Use the available tools to:
1. Get PR details and metadata
2. Get the complete diff
3. Search the codebase for related patterns
4. Get project coding standards

Provide a comprehensive review covering ALL five aspects."""

    if partial_review:
        base_prompt += f"""

⚠️ **Note**: This is a large PR. Reviewing {len(files_to_review)} priority files.
Skipped {len(files_skipped)} non-critical files: {', '.join(files_skipped[:5])}{'...' if len(files_skipped) > 5 else ''}"""

    return base_prompt
```

class PRReviewRequest(BaseModel):
    """Request body for PR review."""
    repo_full_name: str = Field(..., description="Repository in owner/repo format")
    project_id: str = Field(..., description="Pilot Space project UUID")

@router.post("/prs/{pr_number}/review")
async def stream_pr_review(
    pr_number: int,
    request: PRReviewRequest,
    current_user: User = Depends(get_current_user),
    orchestrator: SDKOrchestrator = Depends(get_sdk_orchestrator),
) -> StreamingResponse:
    """Stream PR review via SSE.

    Performs comprehensive review covering 5 aspects (DD-006):
    - Architecture
    - Security
    - Code Quality
    - Performance
    - Documentation
    """
    return StreamingResponse(
        create_sse_stream(
            orchestrator.agents["pr_review"].execute(
                input_data=PRReviewInput(
                    pr_number=pr_number,
                    repo_full_name=request.repo_full_name,
                    project_id=request.project_id,
                    user_id=str(current_user.id),
                    workspace_id=str(current_user.workspace_id),
                ),
                user_id=str(current_user.id),
                workspace_id=str(current_user.workspace_id),
            ),
            event_type="review",
        ),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "Connection": "keep-alive",
            "X-Accel-Buffering": "no",
        },
    )
```
