**Step 2: Create `frontend/src/features/artifacts/utils/docx-purify-config.ts`.**

This is a dedicated DOMPurify config for DOCX-to-HTML output. It MUST differ from HtmlRenderer's PURIFY_CONFIG:
- `ALLOWED_URI_REGEXP: /^(?:https?|mailto|data):/i` — blocks `javascript:` URI scheme in href/src attributes
- `FORBID_TAGS: ['script', 'object', 'embed', 'link', 'base', 'meta', 'form']` — no `'style'` tag in forbid list (docx-preview uses `<style>` blocks for formatting)
- `FORBID_ATTR: ['onerror', 'onload', 'onclick', 'onmouseover']` — block event handlers but NOT `'style'` attribute (mammoth uses inline styles for fonts/colors/indentation)
- Export as `DOCX_PURIFY_CONFIG` constant

**Step 3: Create `frontend/src/features/artifacts/components/renderers/DocxRenderer.tsx`.**

Component signature:
```typescript
interface DocxRendererProps {
  content: ArrayBuffer;
  filename: string;
}
```

Implementation approach:
1. Use `'use client'` directive at top.
2. State: `renderedHtml: string | null`, `renderMode: 'docx-preview' | 'mammoth' | null`, `isRendering: boolean`, `error: string | null`.
3. Primary render path (docx-preview):
   - Import docx-preview: `const { renderAsync } = await import('docx-preview')`
   - Create a container ref (`useRef<HTMLDivElement>`)
   - Call `renderAsync(content, containerRef.current, undefined, { inWrapper: true, ignoreLastRenderedPageBreak: false })` inside a useEffect
   - docx-preview renders directly into the DOM container — it produces its own `<style>` tags and HTML
   - Wrap the container in a sandboxed iframe for style isolation (per CONTEXT.md decision: "Render into sandboxed iframe (same pattern as HtmlRenderer)")
   - For iframe approach: after docx-preview renders into a temporary div, extract the innerHTML and set it as iframe srcDoc
   - Alternative simpler approach: render docx-preview into a `<div>` with `className="docx-preview-container"` and scope styles. If style leakage is observed, switch to iframe.
   - Set `renderMode = 'docx-preview'` on success

4. Fallback render path (mammoth):
   - If docx-preview throws, catch the error and fall back to mammoth
   - `const mammoth = await import('mammoth')`
   - `const result = await mammoth.convertToHtml({ arrayBuffer: content })`
   - Sanitize mammoth HTML: `DOMPurify.sanitize(result.value, DOCX_PURIFY_CONFIG)`
   - Render sanitized HTML in a sandboxed iframe (same pattern as HtmlRenderer: `<iframe srcDoc={sanitizedHtml} sandbox="" />`)
   - Set `renderMode = 'mammoth'` on success
   - No "using fallback" banner shown to user (per CONTEXT.md: "mammoth fallback should be invisible to the user")

5. Page-break indicators (DOCX-03):
   - docx-preview renders page breaks as elements with CSS class. After render, query the container for page-break elements and inject a styled horizontal rule with "Page break" label.
   - For docx-preview: use CSS to style `.docx-preview-container [style*="page-break"]` or the class docx-preview uses for page breaks. The exact class name varies — check rendered output. Common: elements with `style="page-break-before: always"` or `style="page-break-after: always"`.
   - Add CSS: `[style*="page-break-before"], [style*="page-break-after"] { border-top: 1px dashed hsl(var(--border)); margin: 2rem 0; padding-top: 1rem; position: relative; }` with a `::before` pseudo-element showing "Page break" text.
   - For mammoth fallback: mammoth does not preserve page breaks — show continuous flow (acceptable per CONTEXT.md).

6. Error state: If both docx-preview AND mammoth fail, show `<DownloadFallback filename={filename} signedUrl="" reason="error" />`.

7. Loading state: Show a spinner while `isRendering` is true.

**Key security rules:**
- NEVER use HtmlRenderer's PURIFY_CONFIG — it blocks `style` attributes needed for DOCX formatting
- ALWAYS sanitize mammoth output through DOMPurify with DOCX_PURIFY_CONFIG before DOM insertion
- Pin mammoth >= 1.11.0 (already done in install step)
- docx-preview output in sandboxed iframe provides XSS isolation without needing DOMPurify on its output
