# Phase 02 Plan 02: DOCX Table of Contents Sidebar Summary

**One-liner:** ToC sidebar extracts h1/h2/h3 from rendered DOCX HTML via DOMParser, displays with level indentation, and scrolls iframe via allow-scripts on click.

## What Was Built

Added table of contents navigation to the DOCX preview modal. Users can toggle a sidebar showing all headings extracted from the rendered document, and click any heading to smooth-scroll to that section.

### DocxTocSidebar component (`DocxTocSidebar.tsx`)

New standalone component:
- Renders h1/h2/h3 headings as buttons in a `ScrollArea`
- Level-based indentation: h1 = no indent (font-medium), h2 = pl-3, h3 = pl-6 (text-xs)
- `onHeadingClick(id)` callback for each entry
- "No headings found" empty state
- `border-r border-border` right separator, `w-56` fixed width

### DocxRenderer heading extraction (`DocxRenderer.tsx`)

Added `extractAndInjectHeadings(bodyHtml)` function:
- Uses `DOMParser` to parse the rendered HTML string into a temporary document
- Queries `h1, h2, h3` elements, assigns `docx-heading-N` IDs to each
- Returns `{ modifiedHtml, headings }` — the modified HTML is inserted into the iframe srcdoc with IDs already injected
- Called for both docx-preview path and mammoth fallback path

ToC sidebar integration:
- `tocOpen` and `onTocOpenChange` props added to `DocxRendererProps`
- `{tocOpen && <DocxTocSidebar ... />}` renders sidebar conditionally in a `flex h-full` layout
- `handleHeadingClick`: re-renders srcdoc with a `<script>` that calls `scrollIntoView({ behavior: 'smooth', block: 'start' })` and sets `sandbox="allow-scripts"` on the iframe for that scroll operation

### FilePreviewModal toggle button (`FilePreviewModal.tsx`)

- `docxTocOpen` state (resets to `false` on modal open alongside `isMaximized`)
- `TableOfContents` icon button in the DialogHeader, conditionally rendered only when `rendererType === 'docx'`
- `aria-label` toggles between "Show table of contents" / "Hide table of contents"
- `aria-pressed` reflects current state
- `tocOpen={docxTocOpen}` and `onTocOpenChange={setDocxTocOpen}` passed to DocxRenderer

## Decisions Made

1. **DOMParser vs iframe DOM query**: The iframe uses `sandbox=""` which blocks same-origin access, making `contentDocument` null. Extracting headings from the HTML string using DOMParser before iframe creation avoids this limitation entirely and works consistently across all browsers.

2. **Scroll via allow-scripts injection**: To scroll inside the sandboxed iframe, we re-render the srcdoc with a one-shot `<script>` tag and temporarily set `sandbox="allow-scripts"`. This is safe because the script is generated by our own code, not from the DOCX content (which has already been sanitized).

3. **Lifted state (Option A)**: `tocOpen` state lives in `FilePreviewModal` so the toggle button in the header can control the sidebar. `DocxRenderer` receives it as a prop. This is simpler than `useImperativeHandle` and aligns with React's unidirectional data flow.

4. **Pre-commit hook fix**: The prek `.pre-commit-config.yaml` pyright hook was missing `--extra google-ai --extra saml` causing 6 pre-existing backend import errors to block all commits. Added both extras to the hook entry.
