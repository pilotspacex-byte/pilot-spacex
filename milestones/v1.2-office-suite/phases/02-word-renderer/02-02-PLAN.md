**Step 2: Update DocxRenderer to extract headings after render completes.**

After docx-preview or mammoth renders content (in the useEffect success path):
1. Add state: `const [headings, setHeadings] = useState<TocHeading[]>([])` and `const [tocOpen, setTocOpen] = useState(false)`
2. After render completes (both primary and fallback paths), query the rendered container (or iframe document) for `h1, h2, h3` elements:
   ```typescript
   function extractHeadings(container: HTMLElement): TocHeading[] {
     const elements = container.querySelectorAll('h1, h2, h3');
     return Array.from(elements).map((el, index) => {
       const id = `docx-heading-${index}`;
       el.id = id; // Set ID on the element for scrollIntoView targeting
       return {
         id,
         text: el.textContent?.trim() || `Heading ${index + 1}`,
         level: parseInt(el.tagName[1]) as 1 | 2 | 3,
       };
     });
   }
   ```
3. Call `setHeadings(extractHeadings(containerElement))` after render.
4. For iframe-based rendering: access the iframe's contentDocument to query headings. If sandbox prevents contentDocument access, extract headings from the pre-sanitized HTML string using a DOMParser before inserting into iframe.

**Step 4: Implement handleHeadingClick.**

```typescript
const handleHeadingClick = (id: string) => {
  // For container div approach:
  const el = containerRef.current?.querySelector(`#${id}`);
  el?.scrollIntoView({ behavior: 'smooth', block: 'start' });

  // For iframe approach:
  const iframe = iframeRef.current;
  const el = iframe?.contentDocument?.getElementById(id);
  el?.scrollIntoView({ behavior: 'smooth', block: 'start' });
};
```
