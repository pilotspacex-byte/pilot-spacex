   - Add `previous_encrypted_key: Mapped[str | None] = mapped_column(Text, nullable=True, doc="Previous master-encrypted key for dual-key fallback during rotation")`

   - Modify `upsert_key()` to save the current `encrypted_workspace_key` to `previous_encrypted_key` before overwriting with the new key

   - `async def rotate_workspace_key(session: AsyncSession, workspace_id: str, new_raw_key: str, batch_size: int = 100) -> dict[str, int]`:

     4. Query all notes in workspace with non-null body (encrypted content)
     5. Batch re-encrypt: for each batch, decrypt with old key, encrypt with new key, update row
     6. After all batches complete, clear_previous_key
     7. Return {"notes": N, "issues": M} counts of re-encrypted items

   - Add a test for `decrypt_content_with_fallback` — primary succeeds, fallback not tried; primary fails, fallback succeeds; both fail raises InvalidToken

PITFALL: Don't hold a single transaction across all rows. Process in batches with per-batch commits. The batch_size parameter controls this.
