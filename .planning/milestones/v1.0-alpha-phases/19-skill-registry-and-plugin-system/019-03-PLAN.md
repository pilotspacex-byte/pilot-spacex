Create the REST router. Use direct instantiation pattern (NOT @inject DI — follows SCIM/related-issues pattern to avoid wiring_config.modules update).

Router prefix: `/workspaces/{workspace_id}/plugins`
Use `""` (empty string) for root collection routes, NOT `"/"`.

**Endpoints:**

`GET ""` — list installed plugins:
- Requires admin auth (check existing workspace admin auth pattern from workspace_role_skills router)
- Returns `list[WorkspacePluginResponse]`

`GET "/browse"` — browse a GitHub repo:
- Query param: `repo_url: str`
- Decrypt workspace PAT if available; use system GITHUB_TOKEN otherwise
- Call `GitHubPluginService.list_skills()` — block with 502 if GitHub unreachable (no cached fallback)
- Returns `list[SkillListItem]`

`POST ""` — install a plugin:
- Body: `WorkspacePluginInstallRequest`
- Fetch skill content from GitHub, call `InstallPluginService.install()`
- Plugin is stored with is_active=True; SKILL.md is auto-wired immediately
- Any `mcp_tools` / `action_buttons` parsed from SKILL.md frontmatter are stored in the plugin record but NOT wired — Phase 17 (action button data model) is a prerequisite for that wiring
- Returns `WorkspacePluginResponse`

`DELETE "/{plugin_id}"` — uninstall:
- Soft-delete the plugin
- Returns 204

`GET "/check-updates"` — check for updates:
- For each installed plugin: check Redis cache for HEAD SHA, fall back to GitHub API
- Cache result for 5 min: `f"plugin:head_sha:{workspace_id}:{owner}:{repo}"`
- Compare `head_sha != plugin.installed_sha` → `has_update = True`
- Returns `WorkspacePluginUpdateCheckResponse`

`POST "/github-credential"` — save workspace GitHub PAT:
- Body: `WorkspaceGithubCredentialRequest` (raw PAT)
- `pat_encrypted = encrypt_api_key(body.pat)` — single arg
- Upsert via `WorkspaceGithubCredentialRepository`
- Returns `WorkspaceGithubCredentialResponse(has_pat=True)`

`GET "/github-credential"`:
- Returns `WorkspaceGithubCredentialResponse(has_pat=credential is not None)`
