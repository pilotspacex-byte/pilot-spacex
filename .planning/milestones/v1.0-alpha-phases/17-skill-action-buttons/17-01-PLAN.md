    - GET /action-buttons/admin returns all buttons (including inactive) for admins
    - POST /action-buttons creates a button (admin only), returns 201
    - PATCH /action-buttons/{id} updates a button (admin only)
    - PUT /action-buttons/reorder accepts ordered list of button IDs (admin only)
    - DELETE /action-buttons/{id} soft-deletes a button (admin only)
    - Non-admin receives 403 on write endpoints
    - Plugin install auto-creates action buttons from metadata
    - Plugin uninstall deactivates associated action buttons
  </behavior>
  <action>
    1. Create `workspace_action_buttons.py` router following `workspace_plugins.py` pattern exactly:
       - Router prefix: mounted at `/workspaces/{workspace_slug}/action-buttons` in main.py
       - Use direct instantiation (not DI container) — lazy imports inside endpoint functions
       - Admin guard: copy `_require_admin()` from workspace_plugins.py (query WorkspaceMember.role for ADMIN/OWNER)
       - Use `""` (empty string) for collection root routes, NOT `"/"`

       Endpoints:
       - `GET ""` — list active buttons for all members (no admin check). Returns `list[SkillActionButtonResponse]`. Calls `repo.get_active_by_workspace(workspace_id)`.
       - `GET "/admin"` — list ALL buttons (admin only). Returns `list[SkillActionButtonResponse]`. Calls `repo.get_all_by_workspace(workspace_id)`.
       - `POST ""` — create button (admin only). Accepts `SkillActionButtonCreate`. Creates `SkillActionButton` entity, sets `workspace_id`. Returns 201 + `SkillActionButtonResponse`.
       - `PATCH "/{button_id}"` — update button (admin only). Accepts `SkillActionButtonUpdate`. Loads button via `repo.get_by_id()`, applies non-None fields, calls `repo.update()`. Returns `SkillActionButtonResponse`.
       - `PUT "/reorder"` — reorder buttons (admin only). Accepts `SkillActionButtonReorder`. For each button_id in list, set sort_order = index * 10. Flush. Return 204.
       - `DELETE "/{button_id}"` — delete button (admin only). Loads via `repo.get_by_id()`, calls `repo.soft_delete()`. Returns 204.
       - All routes include `session: DbSession` parameter for ContextVar population.

    2. Register router in `main.py`:
       ```python
       from pilot_space.api.v1.routers.workspace_action_buttons import router as action_buttons_router
       # Mount under workspace prefix
       app.include_router(action_buttons_router, prefix="/api/v1/workspaces/{workspace_slug}/action-buttons", tags=["action-buttons"])
       ```
       Follow the exact mounting pattern used for workspace_plugins router.
