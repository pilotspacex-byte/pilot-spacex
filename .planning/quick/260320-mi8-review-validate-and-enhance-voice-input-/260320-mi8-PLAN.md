**useVoiceRecording.ts — Bug fixes:**

1. **Cancel cleanup bug**: In `cancelRecording()`, after setting `onstop = null` and calling `.stop()`, the function calls `cleanupMedia()` which clears `maxDurationTimerRef` — this is correct. BUT the `cleanupMedia` also sets `mediaRecorderRef.current = null` — verify this doesn't race with the `.stop()` call. Actually, the real bug: `cancelRecording` does NOT clear the error auto-reset timeout. Add a ref `errorResetTimerRef` for the `setTimeout` in `setErrorWithAutoReset` and clear it in `cleanupMedia` and on unmount.

2. **audioUrl not reset on new recording**: In `startRecording()`, add `setAudioUrl(null)` alongside the existing `setDurationMs(0)` reset so that stale audio URLs from a previous recording don't persist.

3. **Error auto-reset timeout leak**: The `setTimeout` in `setErrorWithAutoReset` creates a timer that is never cleaned up on unmount. Store it in a ref (`errorResetTimerRef`) and clear it in the unmount cleanup effect.

**useVoiceRecording.ts — Amplitude data enhancement:**

4. Add a new state `amplitudeLevel: number` (0.0 to 1.0) to the hook's return. During recording, create an `AnalyserNode` from an `AudioContext` connected to the media stream. Sample the amplitude every ~100ms using `requestAnimationFrame` or `setInterval` and update `amplitudeLevel`. This enables the RecordButton to show real-time audio visualization. Clean up the AudioContext and AnalyserNode in `cleanupMedia`.

Implementation detail for amplitude:
- Create `audioContextRef = useRef<AudioContext | null>(null)` and `analyserRef = useRef<AnalyserNode | null>(null)`
- In `startRecording`, after getting the `stream`, create: `const audioCtx = new AudioContext(); const analyser = audioCtx.createAnalyser(); analyser.fftSize = 256; const source = audioCtx.createMediaStreamSource(stream); source.connect(analyser);`
- Start an animation frame loop that reads `analyser.getByteTimeDomainData(dataArray)` and computes RMS amplitude normalized to 0-1 range.
- Store the `requestAnimationFrame` ID in a ref and cancel it in `cleanupMedia`.
- In `cleanupMedia`, also close the AudioContext: `audioContextRef.current?.close()`.

5. Extend the return type: add `amplitudeLevel: number` to `UseVoiceRecordingResult`. Initialize to 0, set during recording, reset to 0 on stop/cancel.

**useVoiceRecording.ts — Callback signature change:**

6. Change the `onTranscript` callback signature to `(text: string, audioUrl: string | null) => void` so the parent (ChatInput) receives the audio URL alongside the transcript text. Update the call site in `recorder.onstop` to pass `result.audioUrl` as the second argument.

Update `UseVoiceRecordingOptions.onTranscript` type accordingly.

**RecordButton.tsx — UX enhancements:**

7. **Inline duration display**: When `isRecording`, show the elapsed time `formatDuration(durationMs)` as a small text label next to the stop button, not just in the tooltip. Use `text-xs text-red-500 font-mono tabular-nums` styling.

9. **Visible cancel button**: When `isRecording`, show a small "X" button (using lucide `X` icon, size 3) adjacent to the stop button. Clicking it calls `cancelRecording()`. This replaces the Escape-only cancel UX with a visible affordance.

**AudioPlaybackPill.tsx — New component:**

Create a compact inline audio player pill that renders as a small chip (similar to AttachmentChip in style). Design specs:

- **Layout**: Rounded-full pill, `h-7 px-2.5 gap-1.5 inline-flex items-center` with `bg-muted/50 border border-border/40 rounded-full hover:bg-muted/80 transition-colors`.
- **Play/Pause button**: Small icon button using lucide `Play` (size 3) and `Pause` (size 3). Toggle between states.
- **Progress**: A thin progress bar (`h-0.5 w-16 bg-muted-foreground/20 rounded-full overflow-hidden`) with a fill div (`bg-primary h-full transition-[width]`) that shows playback progress as percentage.

- **Implementation**: Use the HTML5 `<audio>` element with a ref. On play/pause click, toggle `audioRef.current.play()` / `.pause()`. Listen to `timeupdate` event to update progress. Listen to `ended` event to reset to start. Listen to `loadedmetadata` to get duration.
- **Props**: `interface AudioPlaybackPillProps { audioUrl: string; durationSeconds?: number | null; className?: string; onRemove?: () => void; }`. If `onRemove` is provided, show a small X button at the end of the pill to dismiss.

- **Accessibility**: `aria-label="Play recorded audio"` / `"Pause recorded audio"` on the button. The audio element should have `preload="metadata"`.
- **Signed URL expiry**: The signed URL has a 1-hour expiry. If playback fails mid-stream, show a toast: "Audio link expired — re-record to play again".
