**Add confirmation dialog before role changes (Improvement 6):**

In `members-page.tsx`, modify the `handleRoleChange` function (lines 136-153) to show a confirmation dialog before executing the role change, similar to how `handleRemoveMember` works.

Replace the current `handleRoleChange`:

```typescript
const handleRoleChange = async (userId: string, role: WorkspaceRole) => {
  const member = members?.find((m) => m.userId === userId);
  if (!member) return;

  const displayName = member.fullName || member.email;
  const currentRole = member.role;

  // Skip confirmation if role hasn't changed
  if (currentRole === role) return;

  setConfirmDialog({
    open: true,
    title: 'Change member role',
    description: `Change ${displayName}'s role from ${currentRole} to ${role}?`,
    confirmLabel: 'Change Role',
    variant: 'default',
    onConfirm: async () => {
      closeConfirmDialog();
      setUpdatingMemberId(userId);
      const result = await workspaceStore.updateMemberRole(workspaceId, userId, role);
      setUpdatingMemberId(null);

      if (result) {
        toast.success('Role updated', {
          description: `${displayName} is now a ${role}.`,
        });
      } else {
        toast.error('Failed to update role', {
          description: workspaceStore.error ?? 'An unexpected error occurred.',
        });
      }
    },
  });
};
```

This uses the existing `ConfirmActionDialog` and `confirmDialog` state already present in the component (used by `handleRemoveMember` and `handleCancelInvitation`). No new imports or components needed.
