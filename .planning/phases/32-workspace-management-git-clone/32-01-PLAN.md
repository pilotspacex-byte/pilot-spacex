Create `tauri-app/src-tauri/src/commands/workspace.rs` with the following Tauri IPC commands:

**1. `get_projects_dir`** — Returns the current base projects directory path as a String.
- Read `workspace-config.json` from Tauri Store via `StoreExt` (same pattern as auth.rs).
- If "projects_dir" key exists, return that path.
- If absent, compute default: `dirs::home_dir()` + `/PilotSpace/projects/` (use the `dirs` crate).
- Create the directory if it does not exist (`std::fs::create_dir_all`).
- Return the path as a String.

**2. `set_projects_dir`** — Accepts `path: String`, validates it is a directory, persists to `workspace-config.json` Store.
- Verify path exists and is a directory with `std::path::Path::new(&path).is_dir()`.
- If not, return `Err("Path does not exist or is not a directory")`.
- Write to Store: `store.set("projects_dir", serde_json::Value::String(path))` then `store.save()`.
- Return `Ok(())`.

**3. `open_folder_dialog`** — Opens a native folder picker using `tauri_plugin_dialog`.
- Use `tauri_plugin_dialog::DialogExt` to call `app.dialog().file().blocking_pick_folder()`.
- Return `Option<String>` — None if user cancelled, Some(path) if selected.
- IMPORTANT: Use `tauri::async_runtime::spawn_blocking` to wrap the blocking dialog call since it blocks the thread.

**4. `link_repo`** — Accepts `path: String`, validates it is a git repo, adds it to the project list.
- Check `Path::new(&path).join(".git").is_dir()` — if false, return error "Not a git repository".
- Read `workspace-config.json` Store, get or create a "projects" array.
- Each project entry is a JSON object: `{ "name": <dir_name>, "path": <full_path>, "remote_url": <from .git/config>, "linked": true, "added_at": <ISO timestamp> }`.
- Extract remote URL: open `Path::new(&path).join(".git/config")` and parse for `[remote "origin"]` url line, or return "" if no remote.
- Append to projects array. Save Store.
- Return the project entry as serialized JSON.

**5. `list_projects`** — Returns all managed projects from `workspace-config.json` Store.
- Read "projects" array from Store, default to empty array.
- Return `Vec<ProjectEntry>` serialized.

```rust
use std::sync::atomic::{AtomicBool, Ordering};
use std::sync::Arc;
use git2::{Repository, RemoteCallbacks, FetchOptions, build::RepoBuilder, Cred, CredentialType};
use tauri::ipc::Channel;
use tauri_plugin_store::StoreExt;

const KEYCHAIN_SERVICE: &str = "io.pilotspace.app";
const KEYCHAIN_GIT_ACCOUNT: &str = "git_pat";

#[derive(serde::Serialize, Clone)]
pub struct GitProgress {
    pub pct: u32,
    pub message: String,
}

// Module-level static for clone cancellation
static CLONE_CANCEL: std::sync::OnceLock<Arc<AtomicBool>> = std::sync::OnceLock::new();

fn get_cancel_flag() -> Arc<AtomicBool> {
    CLONE_CANCEL.get_or_init(|| Arc::new(AtomicBool::new(false))).clone()
}
```

**1. `git_clone` command:**
```rust
#[tauri::command]
pub async fn git_clone(
    app: tauri::AppHandle,
    url: String,
    target_dir: String,
    on_progress: Channel<GitProgress>,
) -> Result<(), String> {
```
- Reset the cancel flag to `false` at the start.
- Use `tauri::async_runtime::spawn_blocking` because git2 `Repository` is NOT Send — all git2 operations must run on a blocking thread.
- Inside the blocking closure:
  - Create `RemoteCallbacks::new()`.
  - Set `transfer_progress` callback with 2% throttle:
    - Track `last_pct: u32` in a `std::cell::Cell<u32>`.
    - Compute `pct = (stats.received_objects() * 100 / stats.total_objects().max(1)) as u32`.
    - Only send when `pct >= last_pct + 2 || pct == 100`.
    - Check `cancel_flag.load(Ordering::Relaxed)` — if true, return `false` to abort the transfer.
  - Set `credentials` callback with loop detection:
    - Track attempt count in a `std::cell::Cell<u32>`.
    - If `attempts > 3`, return `Err(git2::Error::from_str("Authentication failed after 3 attempts"))`.
    - If `allowed_types.contains(CredentialType::USER_PASS_PLAINTEXT)`:
      - Read PAT from OS keychain: `keyring::Entry::new(KEYCHAIN_SERVICE, KEYCHAIN_GIT_ACCOUNT)`.
      - If found: return `Cred::userpass_plaintext(username, &pat)`.
      - If no PAT in keychain: return `Err(git2::Error::from_str("No credentials configured"))`.
    - Increment attempt count.
  - Build `FetchOptions` with the callbacks.
  - Call `RepoBuilder::new().fetch_options(fo).clone(&url, &PathBuf::from(&target_dir))`.
  - On success: send `GitProgress { pct: 100, message: "Clone complete".into() }`.
- After clone succeeds, add the repo to the projects list in `workspace-config.json` Store:
  - Read existing "projects" array or create empty.
  - Append new `ProjectEntry` with `linked: false`, extract name from target_dir's last component, extract remote_url from the cloned repo, `added_at` as current ISO timestamp.
  - Save Store.
- Return `Ok(())` or the appropriate error.
