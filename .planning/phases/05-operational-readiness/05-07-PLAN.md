<objective>
Fix the runtime wiring bug in the backup CLI: `PilotConfig` has no `database_url` field,
so `backup.py` falls back to passing `config.api_url` (an HTTP REST URL) to `pg_dump` and
`pg_restore`. This causes guaranteed runtime failure because `pg_dump` requires a
`postgresql://` connection string.

Purpose: Satisfy OPS-05 — the backup CLI must actually work at runtime, not just pass
static analysis.

Output: `PilotConfig` gains `database_url` (and `supabase_url`) fields; `backup.py` is
wired to use them; `pilot login` captures both values; CLI command layer has tests.
</objective>
