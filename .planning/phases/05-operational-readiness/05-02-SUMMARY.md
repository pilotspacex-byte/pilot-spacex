## Decisions Made
- `trace_id` aliases the `request_id` generated by `RequestContextMiddleware` — single UUID per request, no extra ID generation needed. Monitoring tools (Datadog, Loki) get both fields for backward-compat.
- `actor` convention: `"user:{uuid}"` — human-readable, filterable with glob patterns in log aggregators.
- `set_action()` is a standalone helper (not a `set_request_context` param) — actions can be set mid-request as business logic proceeds; decoupling from request setup avoids awkward call chains.
- Module-level import of `set_request_context` in auth_middleware (not lazy import) — no circular dependency exists between middleware and logging modules.
