<objective>
Differentiate tablet (768-1024px) and mobile (<768px) sidebar behavior in AppShell, and adapt content area layouts for the tablet viewport.

Purpose: UI-02 requires icon-rail on tablet instead of the current overlay-only approach. UI-03 requires content area adaptation for tablet — reduced margins/padding and stacked layouts where side-by-side would overflow. Currently `isSmallScreen` conflates both viewports into identical overlay behavior.

Output: AppShell shows 60px icon-rail (collapsed inline sidebar) on tablet, overlay drawer on mobile, and full sidebar on desktop. Content area pages render without horizontal overflow at 768-1024px. Settings layout stacks nav + content on tablet.
</objective>

From frontend/src/hooks/useMediaQuery.ts:
```typescript
export function useResponsive(): {
  isMobile: boolean;    // < 768px
  isTablet: boolean;    // 768-1023px
  isDesktop: boolean;   // >= 1024px
  isLargeDesktop: boolean;
  isSmallScreen: boolean; // isMobile || isTablet
  isLargeScreen: boolean;
};
```

Current AppShell uses only `isSmallScreen` from useResponsive. The fix: destructure `{ isMobile, isTablet }` separately.

**Three-mode rendering logic:**
- `isMobile` (< 768px): AnimatePresence overlay drawer (current behavior, keep)
- `isTablet` (768-1024px): Inline `motion.aside` at 60px collapsed width (icon-rail, auto-triggered)
- `isDesktop` (> 1024px): Inline `motion.aside` at full/collapsed width (user-controlled)

Both tablet and desktop use the same inline `motion.aside` — the difference is tablet auto-collapses on mount. The hamburger menu button only shows on `isMobile && !sidebarOpen`.

**NoteCanvasLayout current responsive classes (line 251-253):**
```typescript
'px-4 sm:px-6 md:px-8 lg:px-12 xl:px-16 2xl:px-20'  // horizontal padding
'py-3 sm:py-4 lg:py-6 2xl:py-8'                       // vertical padding
'max-w-full sm:max-w-[640px] md:max-w-[680px] lg:max-w-[720px] xl:max-w-[760px] 2xl:max-w-[800px]'
```
These are already tablet-aware with `md:` prefixes. The main concern is ensuring `min-w-0` is set on flex children so content doesn't overflow when sidebar changes from overlay to icon-rail (gaining ~200px content width on tablet).

**Settings layout (line 247-258):**
```typescript
<div className="flex flex-1 overflow-hidden">
  <nav className="hidden w-56 shrink-0 border-r ... md:block">  // sidebar nav visible at md+
  <div className="flex-1 overflow-y-auto">{children}</div>       // content area
</div>
```
At tablet (768-1024px) with icon-rail sidebar (60px) + settings nav (224px/w-56) + content, space is tight (~484px for content). The settings nav should stack above content at the `md` breakpoint and go side-by-side at `lg`.
