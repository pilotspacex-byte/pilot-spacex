From frontend/src/hooks/useMediaQuery.ts:
```typescript
export function useResponsive(): {
  isMobile: boolean;    // < 768px
  isTablet: boolean;    // 768-1023px
  isDesktop: boolean;   // >= 1024px
  isLargeDesktop: boolean;
  isSmallScreen: boolean; // isMobile || isTablet
  isLargeScreen: boolean;
};
```

Current AppShell uses only `isSmallScreen` from useResponsive. The fix: destructure `{ isMobile, isTablet }` separately.

**Three-mode rendering logic:**
- `isMobile` (< 768px): AnimatePresence overlay drawer (current behavior, keep)
- `isTablet` (768-1024px): Inline `motion.aside` at 60px collapsed width (icon-rail, auto-triggered)
- `isDesktop` (> 1024px): Inline `motion.aside` at full/collapsed width (user-controlled)

Both tablet and desktop use the same inline `motion.aside` — the difference is tablet auto-collapses on mount. The hamburger menu button only shows on `isMobile && !sidebarOpen`.
</interfaces>
