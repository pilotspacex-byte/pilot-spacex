Role claim mapping config format (from CONTEXT.md):
```python
# Stored in workspace.settings["role_claim_mapping"]:
{
    "claim_key": "groups",  # JWT claim key to inspect
    "mappings": [
        {"claim_value": "eng-leads", "role": "admin"},
        {"claim_value": "developers", "role": "member"},
    ]
}
# Rules:
# - claim_key value from JWT is matched against claim_value (exact string or substring of list)
# - Unmapped → always "member" (never "admin" or "owner")
# - claim_value matching is case-insensitive for safety
```

  <action>
    Extend `backend/src/pilot_space/application/services/sso_service.py`:

    Add methods:
    1. `configure_role_claim_mapping(workspace_id, claim_key, mappings, session) -> None`: stores `{"claim_key": claim_key, "mappings": mappings}` in workspace.settings["role_claim_mapping"] via JSONB merge pattern
    2. `get_role_claim_mapping(workspace_id, session) -> dict | None`: returns claim mapping config or None
    3. `map_claims_to_role(claim_value: str | list[str], mapping_config: dict) -> str`:
       - Input: raw claim value from JWT (string or list of strings), workspace mapping config
       - Logic: iterate mappings, compare claim_value case-insensitively against claim_values in config
       - If claim_value is list: return mapped role for first matching item
       - Default: return "member" (never "owner" or "admin" unless explicitly mapped)
       - Validation: if mapping tries to map to "owner", silently cap at "admin" and log warning
    4. `apply_sso_role(user_id, workspace_id, jwt_claims, session) -> WorkspaceMember`:
       - Gets role mapping config for workspace
       - Extracts claim_key value from jwt_claims dict
       - Calls map_claims_to_role
       - Updates workspace_member.role to mapped role (using WorkspaceRole enum)
       - Returns updated member

    Add to `backend/src/pilot_space/api/v1/schemas/sso.py`:
    ```python
    class SsoClaimRoleRequest(BaseModel):
        workspace_id: UUID
        jwt_claims: dict[str, Any]  # claims extracted from Supabase JWT by frontend

    class RoleClaimMappingConfig(BaseModel):
        claim_key: str
        mappings: list[dict[str, str]]  # [{"claim_value": "...", "role": "..."}]
    ```

    Add to `backend/src/pilot_space/api/v1/routers/auth_sso.py`:
    - `POST /auth/sso/claim-role` — requires valid JWT (authenticated user); body: SsoClaimRoleRequest; calls sso_service.apply_sso_role(); returns 200 with applied role name
    - `POST /auth/sso/role-mapping` — admin-only; body: RoleClaimMappingConfig; stores mapping config; returns 200
    - `GET /auth/sso/role-mapping` — admin-only; returns current role claim mapping config
