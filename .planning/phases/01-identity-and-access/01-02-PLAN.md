    Endpoints:
    1. `POST /auth/sso/saml/config` — admin-only; body: SamlConfigRequest; calls sso_service.configure_saml(); returns SamlConfigResponse with metadata_url and acs_url (derived from settings.BACKEND_URL)
    2. `GET /auth/sso/saml/config` — admin-only; returns current SAML config (never expose private key or stored cert in response, only entity_id + sso_url)
    3. `GET /auth/sso/saml/initiate` — query param: workspace_id (UUID); no auth required (unauthenticated users trigger this); calls sso_service.get_saml_config() then saml_provider.get_login_url(); returns SsoInitiateResponse with redirect_url
    4. `POST /auth/sso/saml/callback` — no auth required; receives form body with SAMLResponse; calls saml_provider.process_response(); calls sso_service.provision_saml_user(); returns Supabase JWT as JSON (access_token, token_type); handle SamlValidationError → 401 with detail
    5. `GET /auth/sso/saml/metadata` — no auth required; query param: workspace_id; returns XML response (media_type="application/xml"); calls saml_provider.get_metadata_xml()
    6. `POST /auth/sso/oidc/config` — admin-only; body: OidcConfigRequest; calls sso_service.configure_oidc(); returns OidcConfigResponse
    7. `GET /auth/sso/oidc/config` — admin-only; returns current OIDC config
    8. `PATCH /auth/sso/enforcement` — admin-only; body: `{"sso_required": bool}`; calls sso_service.set_sso_required(); returns 204
    9. `GET /auth/sso/status` — NO auth required; query param: workspace_id (UUID); calls sso_service.get_sso_status(); returns SsoStatusResponse with {has_saml, has_oidc, sso_required, oidc_provider}. This endpoint is called by the login page before the user is authenticated to determine whether to show an "SSO Login" button.
