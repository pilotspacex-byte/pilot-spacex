<objective>
Close two runtime blockers preventing end-to-end SSO from working:

1. Bug 1 (422 on admin endpoints): All admin SSO endpoints accept `workspace_id: UUID` but the frontend sends `workspace_slug` (string). FastAPI rejects non-UUID strings with 422. Fix: add `_resolve_workspace` slug→UUID helper (same pattern as `audit.py`) and change admin endpoint signatures from `workspace_id: UUID` to `workspace_slug: str`.

2. Bug 2 (no JWT after SAML login): The SAML callback returns `{"token_type": "saml_provisioned"}` — a placeholder. The frontend has no handler for this. Fix: call `admin.generate_link(type="magiclink")` in `provision_saml_user`, redirect to `/auth/saml-callback?token_hash=...`, and add a frontend page that calls `supabase.auth.verifyOtp` to exchange the hash for a real JWT session.

Purpose: AUTH-01 through AUTH-04 are structurally implemented but non-functional at runtime due to these two parameter/protocol mismatches.
Output: Functional end-to-end SSO — admin config endpoints accept slugs, SAML login produces a real Supabase JWT session.
</objective>

```python
async def _resolve_workspace(workspace_slug: str, session: AsyncSession) -> UUID:
    workspace_repo = WorkspaceRepository(session)
    try:
        as_uuid = UUID(workspace_slug)
        workspace = await workspace_repo.get_by_id_scalar(as_uuid)
    except ValueError:
        workspace = await workspace_repo.get_by_slug_scalar(workspace_slug)
    if workspace is None:
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail="Workspace not found")
    return workspace.id
```
