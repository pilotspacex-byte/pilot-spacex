  <action>
Audit page-level layout containers in the files listed above. For each file:

1. **Find structural padding/margin classes** — page headers, content wrappers, section dividers.
2. **Replace non-8px-grid values with 8px-grid values:**
   - `py-3` (12px) -> `py-2` (8px) or `py-4` (16px) — choose whichever preserves visual balance
   - `py-5` (20px) -> `py-4` (16px) or `py-6` (24px)
   - `px-3` (12px) -> `px-2` (8px) or `px-4` (16px)
   - `px-5` (20px) -> `px-4` (16px) or `px-6` (24px)
   - `py-1` (4px) is acceptable for tight spacing — leave as-is
   - `px-1` (4px) is acceptable for tight inline spacing — leave as-is
   - `gap-3` (12px) -> `gap-2` (8px) or `gap-4` (16px)
   - `gap-5` (20px) -> `gap-4` (16px) or `gap-6` (24px)
   - `space-y-3` -> `space-y-2` or `space-y-4`
3. **Check for hardcoded color overrides** — `bg-[#...]`, `text-[#...]`, `border-[#...]`, or `style={{ color: '...' }}`. Replace with token-based Tailwind classes (e.g., `bg-background`, `text-foreground`, `border-border`). If a hardcoded value is for a specific non-token purpose (e.g., inline chart colors), leave it with a comment explaining why.

**Scope limitation per research:** Only touch page-level layout files in `frontend/src/app/`. Do NOT modify internal component spacing in `frontend/src/components/` or `frontend/src/features/` — those are shadcn/ui primitives and feature components that should not be changed in bulk.

Prefer `py-4` over `py-2` when the current `py-3` is in a page header context (headers need breathing room). Prefer `py-2` when current `py-3` is in a compact toolbar context.
  </action>
