<objective>
Make RateLimitMiddleware active at runtime by fixing the architectural gap in two steps:

1. Refactor RateLimitMiddleware to use a lazy Redis accessor (same pattern as SessionRecordingMiddleware) so it can be registered at module level without requiring a connected Redis client at import time.
2. Move app.add_middleware(RateLimitMiddleware) from the commented-out lifespan block to module level in main.py.
3. Add an integration test that uses TestClient (not dispatch()) to verify the middleware is active and returns 429.

Purpose: Phase 06 registered the middleware inside the lifespan, which is called after the middleware stack is frozen — the middleware was never active. Phase 11 closes this architectural gap so TENANT-03 rate limiting is enforced at runtime.
Output: Modified rate_limiter.py (lazy accessor), modified main.py (module-level registration), updated test file (integration test proving 429 fires).
</objective>

```python
class SessionRecordingMiddleware(BaseHTTPMiddleware):
    def __init__(self, app) -> None:
        super().__init__(app)
        self._redis: RedisClient | None = None
        self._session_factory: object | None = None

    def _resolve_dependencies(self, request: Request) -> None:
        """Lazily resolve redis and session_factory from app container."""
        if self._redis is None or self._session_factory is None:
            try:
                container = request.app.state.container
                self._redis = container.redis_client()
                self._session_factory = container.session_factory()
            except Exception:
                logger.debug("SessionRecordingMiddleware: container not yet available")
```

    ```python
    class TestRateLimitMiddlewareMainRegistration:
        """Verify RateLimitMiddleware is registered in main.app and returns 429."""

        @pytest.mark.asyncio
        async def test_middleware_active_returns_429_via_testclient(self) -> None:
            """RateLimitMiddleware in main.app returns 429 when Redis INCR exceeds limit.

            Uses TestClient (not dispatch()) so the full middleware stack is built.
            Patches _resolve_redis to inject a mock Redis that always returns count > limit.
            Does not require a running Redis instance.
            """
            from unittest.mock import patch, AsyncMock, MagicMock
            from fastapi import HTTPException
            from starlette.testclient import TestClient

            from pilot_space.main import app
            from pilot_space.api.middleware.rate_limiter import RateLimitMiddleware

            # Build a mock Redis where INCR always returns 9999 (exceeds all limits)
            mock_redis_raw = AsyncMock()
            mock_redis_raw.incr = AsyncMock(return_value=9999)
            mock_redis_raw.expire = AsyncMock(return_value=True)
            mock_redis_raw.get = AsyncMock(return_value=None)
            mock_redis_raw.set = AsyncMock(return_value=True)

            def fake_resolve_redis(self_mw, request):
                # Inject mock Redis directly, skip container lookup
                self_mw.redis = mock_redis_raw
                self_mw._redis_resolved = True

            with patch.object(RateLimitMiddleware, "_resolve_redis", fake_resolve_redis):
                with TestClient(app, raise_server_exceptions=False) as client:
                    # Verify middleware is in the stack
                    stack = app.middleware_stack
                    found = False
                    for _ in range(15):
                        if "RateLimitMiddleware" in type(stack).__name__:
                            found = True
                            break
                        stack = getattr(stack, "app", None)
                        if stack is None:
                            break
                    assert found, "RateLimitMiddleware not found in app.middleware_stack"

                    # Verify 429 is returned (BaseHTTPMiddleware wraps HTTPException as 500
                    # when raise_server_exceptions=False, but ExceptionMiddleware converts
                    # HTTPException to its status code — 429 should reach the client)
                    response = client.get(
                        "/api/v1/workspaces",
                        headers={"X-Workspace-ID": "test-workspace-id"},
                    )
                    assert response.status_code == 429, (
                        f"Expected 429 from rate limit, got {response.status_code}"
                    )
                    assert "Retry-After" in response.headers
    ```

    Note: `raise_server_exceptions=False` is used because Starlette's `ExceptionMiddleware` converts `HTTPException` to real HTTP responses. The 429 from `_raise_rate_limit_exceeded()` (which raises `HTTPException(status_code=429)`) should become a real 429 HTTP response. If the test reveals it becomes 500, use `raise_server_exceptions=True` and catch `HTTPException` directly via `pytest.raises` after calling `TestClient` outside the context.

    The test must use `from pilot_space.main import app` — NOT create a fresh FastAPI() instance — because the purpose is to verify the real main.py registers the middleware.
